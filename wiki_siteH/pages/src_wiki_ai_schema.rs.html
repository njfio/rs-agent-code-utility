    Overview

    The src/wiki/ai_schema.rs module defines a lightweight, serde‑compatible schema system used throughout the wiki AI tooling. It provides type‑safe representations of field definitions, supports runtime validation, and can be serialized to/from JSON for storage or network transport.

    Deep Dive

    • struct AiSchema
      Root container for a collection of fields describing an AI‑driven document schema.
    • struct Field
      Describes a single attribute within an `AiSchema`.
    • enum DataType
      Enumerates the primitive and composite types supported by the schema engine.
    • method AiSchema::validate
      Runs runtime validation of a JSON payload against the schema definition.

    Key APIs

    • pub fn new(version: impl Into<String>) -> Self
      Creates an empty schema with the supplied version identifier.
    • pub fn add_field(&mut self, field: Field) -> Result<(), SchemaError>
      Inserts a new field; returns an error if a field with the same name already exists.
    • pub fn validate(&self, payload: &serde_json::Value) -> Result<(), Vec<ValidationError>>
      Validates a JSON payload; useful before feeding data to downstream AI models.
    • pub fn to_json(&self) -> serde_json::Value
      Serializes the schema itself to JSON for persistence or transmission.
    • pub fn from_json(v: &serde_json::Value) -> Result<Self, SchemaError>
      Deserializes a previously stored schema representation.

    Examples

    Defining and serializing a simple schema
    use std::collections::HashMap;
use serde_json::json;
use wiki::ai_schema::{AiSchema, Field, DataType};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let mut schema = AiSchema::new("v1");
    schema.add_field(Field::new("title", DataType::String, true))?;
    schema.add_field(Field::new("views", DataType::Integer, false))?;
    let json_schema = schema.to_json();
    println!("Schema JSON: {}", json_schema);
    Ok(())
}

    Validating incoming data
    use serde_json::json;
use wiki::ai_schema::AiSchema;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let schema = AiSchema::from_json(&json!({
        "version": "v1",
        "fields": {
            "title": {"type": "String", "required": true},
            "tags": {"type": {"Array": "String"}, "required": false}
        }
    }))?;

    let payload = json!({"title": "Rust Docs", "tags": ["rust", "wiki"]});
    match schema.validate(&payload) {
        Ok(_) => println!("Payload is valid"),
        Err(errs) => eprintln!("Validation errors: {:?}", errs),
    }
    Ok(())
}

    Security

    Risks
    • An attacker could craft a malicious schema JSON that triggers panic during deserialization (e.g., extremely deep nesting).
    • If schema‑driven validation is bypassed, downstream AI models may receive malformed inputs, leading to model poisoning.
    Mitigations
    • Enforce a maximum nesting depth (configurable via `AiSchema::with_max_depth`).
    • Validate schema JSON against a known whitelist of allowed `DataType` variants before accepting it from external sources.
    • Run schema parsing in a sandboxed thread with a timeout to prevent DoS attacks.

    Performance

    Concerns
    • Validation walks the entire payload tree; large documents can become CPU‑bound.
    Tips
    • Pre‑compute a flattened field lookup table (`HashMap<&str, &Field>`) when constructing the schema to avoid repeated hashmap lookups during validation.
    • Reuse the same `AiSchema` instance across requests – construction is relatively expensive due to hashmap allocations.

    Overview

    The code_migration_assistant.rs example demonstrates a small command‑line utility that helps developers migrate Rust codebases to a new edition or API surface. It parses command‑line arguments, loads a configuration, walks the source tree, uses the syn crate to build an AST, applies a series of MigrationSteps, and finally emits a human‑readable report and optional patch files. The example showcases idiomatic error handling, parallel file processing with rayon, and safe interaction with the file system.

    Examples

    CLI Quick‑Start
    cargo run --example code_migration_assistant -- ./my_project --edition 2021 --dry-run

    Runs the assistant on the my_project directory, targeting the 2021 edition, and prints a report without modifying any files.

    Programmatic Use in a Build Script
    use std::path::PathBuf;
use code_migration_assistant::{MigrationConfig, MigrationStep, run_migration, generate_report};

fn main() -> anyhow::Result<()> {
    let cfg = MigrationConfig::new(PathBuf::from("src"), Edition::2021, false, true)?;
    let steps = vec![
        MigrationStep::RenameCrate { old: "old_lib".into(), new: "new_lib".into() },
        MigrationStep::UpdateMacro { name: "serde_json".into(), version: 2 },
    ];
    let report = run_migration(&cfg, &steps)?;
    generate_report(&report, &cfg.source_dir)?;
    Ok(())
}

    Shows how to embed the migration logic inside a custom build script, applying fixes directly and emitting a markdown report.

    Security

    Risks
    • If `apply_fixes` is enabled, malformed patches could corrupt source files.
    • Path traversal via malicious `exclude_patterns` could cause the tool to read/write outside the intended project directory.
    Mitigations
    • Always perform a `dry_run` first and review the generated `REPORT.md` before applying changes.
    • Validate that all paths resolved from patterns are canonicalized and reside under `config.source_dir`.

    Performance

    Concerns
    • Parsing with `syn` is CPU‑intensive; processing tens of thousands of files can take several minutes.
    • Memory usage grows with the number of concurrently parsed ASTs when using the default parallel iterator.
    Tips
    • Enable `rayon::ThreadPoolBuilder::num_threads` to match the number of physical cores. Use the `--max-files` flag (added in the example) to limit the batch size for very large codebases. Consider re‑using a single `syn::parse_file` buffer when possible to reduce allocations.

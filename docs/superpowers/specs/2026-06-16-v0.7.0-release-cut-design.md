# v0.7.0 Release Cut — Design

**Status:** Approved (design phase)
**Date:** 2026-06-16
**Author:** njf + Claude
**Topic:** Cut the `v0.7.0` release of merged-but-unreleased work

---

## Goal

Ship the work that has merged to `main` since `v0.6.1` but is not yet
released, as a single `v0.7.0` release: markdown/prose indexing,
structural-query grep, and four reliability fixes. Take it all the way
through a published GitHub release with prebuilt tarballs.

## Why a minor bump (`0.6.1 → 0.7.0`)

Under 0.x semver as practiced in this repo, a new **default-on
capability** or a user-visible **behavior change** warrants a minor
bump. Markdown indexing is both: a 13th indexed language, on by
default, so `find_symbol`/`outline_workspace`/`find_callers` now see
`.md` files where they previously did not. The structural-grep CLI
flags are additive; the four fixes are bug fixes. Bundled under the
markdown headline, the release is `0.7.0`.

The **frozen user surface is unchanged**: 10 MCP tools, 10 `rts` CLI
subcommands, same names and argument shapes. New grep flags
(`--structural-query`, `--language`, `--within-symbol`,
`--within-symbol-allow-overload`, `--multiline`) are additive and not
part of the frozen set. **No breaking change.**

The eight merged changes since `v0.6.1`. Five already have

## Out of scope (stays deferred)

macOS Intel prebuilt binaries, Windows port, full macOS notarization,
public agent-bench baseline, streaming-response cancellation, overload
disambiguation, Sentry telemetry. None block this release.

## Procedure

Mirrors the `v0.6.0` cut: one `release/v0.7.0` branch, one bundled
release commit, one PR.

### 1. Backfill changelog fragments

Add three fragments under `changelog.d/`, following
`changelog.d/README.md` (top-level `###` header, no front-matter):

- `145-feat-structural-query-grep.md` — `rts grep --structural-query`
  with tree-sitter node-kind filtering (`--language` required;
  `--within-symbol`, `--multiline` companions). The grep plain text
  can't express: "string literals containing X", "identifier usages of
  Y".
- `148-fix-callers-exclude-prose.md` — `find_callers` / `impact_of` no
  longer surface prose/markdown mentions of a name as call sites
  (identifier-regex fallback skipped for Markdown).
- `149-fix-structural-limit-bounds-filtered.md` — structural grep
  `--limit` bounds the **returned** match set, not the raw pre-filter
  node scan, so a match past the first N nodes is still found.

### 2. Roll up the changelog

`scripts/build-changelog.sh 0.7.0 --dry-run` first to preview, then
`scripts/build-changelog.sh 0.7.0`. This inserts
`## [0.7.0] - 2026-06-16` between `## [Unreleased]` and `## [0.6.1]`,
concatenating fragments sorted by filename, and deletes the consumed
fragments. Hand-polish the resulting section so markdown indexing leads
and its **behavior change** (`.md` now indexed by default) is called
out explicitly for upgraders.

### 3. Version bump

- Root `Cargo.toml`: `version = "0.6.1"` → `"0.7.0"` (line 24;
  workspace-wide — all crates inherit via `version.workspace = true`).
- Refresh `Cargo.lock` (`cargo build` regenerates the workspace
  package versions).

### 4. README + docs version sweep

- `README.md:65` install `VERSION=0.6.1` → `0.7.0`.
- `README.md:124` `Latest release: v0.6.1` → `v0.7.0`; the "stable for
  daily use" line moves from **v0.6** to **v0.7** framing.
- `README.md:126-130` — reframe the "**Landing on `main` (→ v0.7),
  build from source to try today**" block as **shipped in v0.7.0**
  (these features are now released, not main-only).
- `README.md` languages badge (line 4) and prose (line 120) `12` →
  `13`; add Markdown to the indexed-language list with a note that it's
  prose, not call-edge-bearing.
- `README.md` grep tool row (line 97) — mention `--structural-query`.
- `docs/install.md:50` `VERSION=0.6.1` → `0.7.0`.
- **Do not** change `README.md:67` `v0.6.1+` — it is a "feature
  available since" marker for `gh attestation`, not a current-version
  reference.
- Sweep for any other literal `0.6.1` current-version strings
  (`git grep -n "0\.6\.1"`); leave historical/`CHANGELOG.md` references
  intact.

### 5. Validate locally

- `cargo fmt --all --check` (enforced in CI — must pass; never mask
  exit code behind a pipe).
- `cargo build --workspace --release`.
- `cargo test --workspace`.
- `cargo clippy --workspace --all-targets` (advisory).
- Confirm `target/release/rts --version` prints `0.7.0`.

### 6. PR

- Branch `release/v0.7.0`, single commit bundling: backfilled
  fragments' deletion + CHANGELOG `[0.7.0]` section + `Cargo.toml` /
  `Cargo.lock` bump + README/docs sweep.
- Commit message ends with the `Co-Authored-By: Claude Opus 4.8`
  trailer; PR body ends with the 🤖 footer.
- Open PR `release: v0.7.0`, wait for CI green (Rust CI + protocol-v0
  schema-drift), resolve any review comments, merge to `main`.

### 7. Tag and build

- After merge, on updated `main`: `git tag v0.7.0` and push the tag.
- `release.yml` (trigger `push tag v*`) builds three tarballs
  (`x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`,
  `aarch64-apple-darwin`) and uploads them as a **draft** release.

### 8. Verify and publish

- Wait for the release workflow to go green; confirm all three tarballs
  + `SHA256SUMS` are attached to the draft.
- Spot-check: download one tarball, confirm it unpacks and the binary
  reports `0.7.0`.
- Flip the draft to published (`gh release edit v0.7.0 --draft=false`,
  or the prepared release notes from the CHANGELOG section).

## Risks and mitigations

- **Naive rollup omits #145/#148/#149.** Mitigated by step 1 (backfill
  before rollup). This is the single most important step.
- **`fmt` exit code masked by a pipe** (a prior near-miss). Run
  `cargo fmt --all --check` standalone; never `| head`.
- **Tag triggers a public-facing build.** The workflow uploads a
  *draft*; nothing is public until step 8. Confirm artifacts before the
  publish flip.
- **`build-changelog.sh` idempotency guard** refuses if `0.7.0`
  already appears in `CHANGELOG.md` — so re-runs are safe; if it
  aborts, the section already exists.
- **Cargo.lock drift in CI.** Bumping the workspace version changes
  `Cargo.lock`; commit it so CI's locked build matches.

## Success criteria

- `v0.7.0` published on GitHub with three prebuilt tarballs +
  `SHA256SUMS`.
- `CHANGELOG.md` has a complete `## [0.7.0]` section covering all eight
  changes (no omissions); `changelog.d/` holds only `README.md`.
- `main` builds at `0.7.0`; `rts --version` == `0.7.0`.
- README and `docs/install.md` reference `0.7.0`; no stale `0.6.1`
  current-version strings; languages count is `13`.
- CI green on the release PR and the release workflow.

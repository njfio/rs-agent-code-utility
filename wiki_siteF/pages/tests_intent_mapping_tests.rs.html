    Overview

    This file contains integration tests for the intent mapping subsystem. The tests verify that raw user intents are correctly transformed into internal representations, that edge‑cases are handled gracefully, and that error paths return appropriate diagnostics. The suite exercises the public API of src/intent_mapping.rs using realistic JSON payloads and mock services.

    Deep Dive

    • test test_intent_mapping_success
      Ensures a well‑formed intent JSON is mapped to the correct internal struct.
    • test test_intent_mapping_missing_fields
      Verifies that missing mandatory fields produce a descriptive error.
    • test test_intent_mapping_invalid_type
      Checks type‑validation for numeric fields.
    • test test_intent_mapping_unknown_action
      Ensures unknown actions are flagged as unsupported.
    • test test_intent_mapping_bulk
      Runs a batch of intents to validate performance and stability.

    Examples

    Basic mapping of a valid intent
    let json = r#"{ "action": "create", "parameters": { "name": "example" }, "confidence": 0.95 }"#;

assert_eq!(intent.action, Action::Create);

    Demonstrates the happy‑path usage of map_intent with a minimal JSON payload.

    Handling a missing field error
    let json = r#"{ "parameters": {}, "confidence": 0.8 }"#;
match map_intent(json) {
    Err(IntentError::MissingField(field)) => println!("Missing field: {}", field),
    _ => panic!("unexpected result"),
}

    Shows how to pattern‑match on IntentError to provide user‑friendly diagnostics.

    Security

    Risks
    • If a test loads untrusted JSON it could trigger a panic in `serde_json` leading to test suite aborts. Large payloads may cause excessive memory allocation during bulk tests.
    Mitigations
    • All test JSON files are version‑controlled and reviewed. Bulk tests cap the payload size to 1 KB and enforce a total memory budget via `RUST_TEST_THREADS=1` during CI.

    Performance

    Concerns
    • Bulk test iterates 1,000 times; on low‑end CI runners this may exceed default test timeout. Repeated parsing of identical strings can be optimized by reusing a pre‑parsed `serde_json::Value` in benchmarks.
    Tips
    • Use `cargo test -- --test-threads=1` to get deterministic timing. Consider `criterion` benchmarks for more granular performance analysis beyond the simple bulk test.

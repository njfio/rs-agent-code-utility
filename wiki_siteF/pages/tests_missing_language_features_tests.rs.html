    Overview

    The tests/missing_language_features_tests.rs file contains a suite of integration tests that verify the compiler's handling of language features that are not yet stabilized or are deliberately omitted in certain configurations. Each test asserts that attempting to use a missing feature results in the expected compilation error, ensuring that feature gating and error messages remain consistent across compiler versions.

    Deep Dive

    • function test_missing_async_await
      Ensures that the `async`/`await` syntax is rejected when the `async_await` feature flag is disabled.
    • function test_missing_const_generics
      Validates that const generic parameters are unavailable without the `const_generics` feature.
    • function test_missing_try_trait
      Checks that the `?` operator cannot be used when the `try_trait` feature is missing.
    • helper function assert_missing_feature
      Utility that runs a code snippet through `rustc` and asserts a specific feature-gating error.
    • helper function run_test_case
      Wraps `trybuild::TestCases::compile_fail` for concise test definitions.

    Key APIs

    • fn assert_missing_feature(code: &str, expected_error: &str)
      assert_missing_feature(r#"async fn foo() {}"#, "E0660");
    • fn run_test_case(path: &str)
      run_test_case("tests/ui/missing_features/async_missing.rs");

    #[test]
fn test_async_missing() {

    Security

    Risks
    • The test suite compiles arbitrary Rust code snippets, which could be exploited if untrusted input is injected into the test harness.
    Mitigations
    • All test snippets are hard‑coded in the repository; never interpolate external data into the code strings.
    • Run the test suite in a sandboxed environment (e.g., `cargo test` inside a CI container) to limit potential damage.

    Performance

    Concerns
    • Compiling many small snippets with `rustc` can be slow, especially on CI runners with limited CPU resources.
    Tips
    • Group related snippets into a single `trybuild` test case to reduce the number of separate compiler invocations.
    • Enable `cargo test --release` for faster compilation of the test harness itself.

    Overview

    This example demonstrates advanced usage of the GPT‑4o API in Rust, covering synchronous and asynchronous calls, streaming responses, multi‑modal inputs (text + image), custom request options, error handling, and best‑practice patterns for client reuse.

    Deep Dive

    • struct Gpt4oClient
      High‑level wrapper around the OpenAI GPT‑4o HTTP endpoint.
    • struct RequestOptions
      Fine‑grained control over model parameters such as temperature, max_tokens, and response_format.
    • struct Response
      Deserialized JSON payload returned by the GPT‑4o service.
    • struct ParsedResult
      Convenient wrapper that extracts the most relevant parts of a Response for downstream processing.
    • enum Model
      Supported GPT‑4o model variants.
    • function init_client
      Factory helper that creates a Gpt4oClient from an API key.
    • method send_prompt
      Sends a text (or text + image) prompt to the model and returns a Response.
    • method stream_prompt
      Streams partial tokens as they become available.

    Key APIs

    • pub fn new(api_key: &str) -> Self
      Creates a client with default options; typically called once at program start.
    • pub async fn send_prompt(&self, prompt: &str, options: Option<RequestOptions>) -> Result<Response, Gpt4oError>
      Core API call; pass None to use defaults or Some(opts) for custom parameters.
    • pub async fn stream_prompt<F>(&self, prompt: &str, options: Option<RequestOptions>, mut on_chunk: F) -> Result<(), Gpt4oError> where F: FnMut(&str) + Send
      Provides token‑by‑token streaming; useful for chat‑like interfaces.
    • pub fn text(&self) -> &str
      Convenient accessor for the generated text portion of a Response.
    • impl Default for RequestOptions
      Yields a baseline configuration (temperature 0.7, max_tokens 1024, etc.).

}

    hits: u64,
    misses: u64,
    evictions: u64,

}

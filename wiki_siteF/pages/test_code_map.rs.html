    Overview

    The test_code_map.rs module provides utilities for creating and querying a lightweight in‑memory mapping between source code identifiers and their metadata. It is primarily used in test suites to simulate code‑analysis scenarios without pulling in heavy compiler infrastructure. The module defines a CodeMap struct, a SymbolKind enum, and several helper functions for insertion, lookup, and iteration over the stored symbols.

    Deep Dive

    • struct CodeMap
      Container that holds a hash map of identifier strings to `SymbolInfo`.
    • struct SymbolInfo
      Metadata associated with a code identifier.
    • enum SymbolKind
      Classification of a code symbol.
    • fn insert
      Insert a single identifier into the map.
    • fn insert_batch
      Insert multiple identifiers at once.
    • fn get
      Retrieve metadata for a given identifier.
    • fn iter_symbols
      Iterate over all stored symbols.

    Key APIs

    • pub fn new() -> Self
      let mut map = CodeMap::new();
    • pub fn insert(&mut self, name: impl Into<String>, info: SymbolInfo) -> Option<SymbolInfo>
      map.insert("my_func", SymbolInfo::new(SymbolKind::Function, 12, None));
    • pub fn get(&self, name: &str) -> Option<&SymbolInfo>
      if let Some(info) = map.get("my_var") { assert_eq!(info.kind, SymbolKind::Variable); }
    • pub fn insert_batch<I>(&mut self, entries: I) where I: IntoIterator<Item=(String, SymbolInfo)>
      let batch = vec![ ("foo".to_string(), info1), ("bar".to_string(), info2) ]; map.insert_batch(batch);
    • pub fn iter_symbols(&self) -> impl Iterator<Item=(&String, &SymbolInfo)>
      for (name, info) in map.iter_symbols() { println!("{}: {:?}", name, info.kind); }

    Examples

    Basic insertion and lookup
    use test_code_map::{CodeMap, SymbolInfo, SymbolKind};

let mut map = CodeMap::new();
let info = SymbolInfo::new(SymbolKind::Function, 42, Some("does something".into()));
map.insert("do_something", info);

assert_eq!(map.get("do_something").unwrap().line, 42);

    Creates a fresh CodeMap, inserts a single function symbol, and verifies that the stored line number matches the expectation.

    Batch insertion and filtering
    use test_code_map::{CodeMap, SymbolInfo, SymbolKind};

let mut map = CodeMap::new();
let entries = vec![
    ("foo".to_string(), SymbolInfo::new(SymbolKind::Variable, 3, None)),
    ("Bar".to_string(), SymbolInfo::new(SymbolKind::Struct, 10, None)),
    ("baz".to_string(), SymbolInfo::new(SymbolKind::Function, 15, None)),
];
map.insert_batch(entries);

let functions: Vec<_> = map.iter_symbols()
    .filter(|(_, info)| info.kind == SymbolKind::Function)
    .collect();
assert_eq!(functions.len(), 1);

    Shows how to insert many symbols at once and then filter the map to retrieve only functions, a common pattern in test validation.

    Security

    Risks
    • The module stores raw strings supplied by test harnesses; if untrusted data is fed into `insert_batch`, it could cause a denial‑of‑service via extremely large keys or many entries.
    Mitigations
    • Limit the size of identifiers (e.g., truncate or reject >256 bytes) before insertion, and cap the total number of entries in a `CodeMap` when used in integration tests.

    Performance

    Concerns
    • Each lookup is O(1) average but degrades to O(n) in the worst case if many keys collide; extremely large maps (>10⁶ entries) may exhibit noticeable slowdown.
    Tips
    • Reserve capacity upfront with `CodeMap::with_capacity(n)` when the expected number of symbols is known, reducing re‑hashing overhead.

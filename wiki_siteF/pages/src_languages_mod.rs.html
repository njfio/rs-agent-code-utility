    Overview

    The src/languages/mod.rs module is the central registry and utility hub for language handling within the project. It defines the Language enum, configuration structures, detection helpers, and runtime registration APIs. This module enables the rest of the codebase to query language‑specific metadata (file extensions, comment syntax, display name) in a thread‑safe, lazy‑initialized fashion.

    Deep Dive

    • enum Language
      Enumerates all built‑in programming languages supported by the crate.
    • struct LanguageConfig
      Holds runtime‑configurable metadata for a language.
    • struct CommentSyntax
      Describes line and block comment delimiters for a language.
    • function detect_language
      Detects the language of a file based on its path and extension.
    • function register_language
      Registers a custom language at runtime.

    Key APIs

    • fn from_extension(ext: &str) -> Option<Language>
      let lang = Language::from_extension("rs"); // Some(Language::Rust)
    • fn name(&self) -> &'static str

    Examples

    Detecting a file's language
    use std::path::Path;
use crate::languages::{detect_language, Language};

fn main() {
    let path = Path::new("src/lib.rs");
    match detect_language(path) {
        Some(Language::Rust) => println!("Rust source detected"),
        Some(other) => println!("Detected: {}", other.name()),
        None => println!("Unknown language"),
    }
}

    The example shows the typical workflow: call detect_language with a Path, match on the returned Language enum, and use name() for a human‑readable label.

    Registering a custom language at runtime
    use crate::languages::{LanguageConfig, register_language, CommentSyntax};

fn add_my_lang() -> Result<(), Box<dyn std::error::Error>> {
    let config = LanguageConfig {
        name: "MyLang".to_string(),
        extensions: vec!["ml".to_string()],
        comment_syntax: Some(CommentSyntax {
            line: Some("#".to_string()),
            block: None,
        }),
    };
    register_language(config)?;
    Ok(())
}

    Custom languages can be injected without recompiling the crate. The function validates uniqueness of the .ml extension before insertion.

    Security

    Risks
    • If a malicious plugin supplies a `LanguageConfig` with a crafted comment delimiter that includes control characters, downstream tools that embed comments could be tricked into code injection.
    • Registering languages from untrusted sources may overwrite built‑in extensions, leading to mis‑classification and potential execution of wrong parsers.
    Mitigations
    • Validate and sanitise all string fields in `LanguageConfig` before insertion – reject control characters and enforce a whitelist of printable ASCII.
    • Make `register_language` return an error on attempts to override built‑in extensions; keep a separate namespace for user‑defined languages.

    Performance

    Concerns
    • The global `LANGUAGES` map is lazily initialised using `once_cell::sync::Lazy`; the first lookup incurs a lock acquisition cost.
    • Repeated calls to `detect_language` on the same path perform redundant hash lookups; caching results at a higher layer can improve throughput.
    Tips
    • If you need to detect many files in a batch, build a local `HashMap<&str, Language>` of extensions once and reuse it.
    • Prefer `Path::extension` over manual string parsing to avoid unnecessary allocations.

      pub fn detect_language_from_path<P: AsRef<std::path::Path>>(path: P) -> Option<Language> {
    let path = path.as_ref();
    let extension = path.extension()?.to_str()?.to_lowercase();

    for language in Language::all() {
        if language.file_extensions().contains(&extension.as_str()) {
            return Some(language);
        }
    }

    None
}

          fn from_str(s: &str) -> Result<Self> {
        match s.to_lowercase().as_str() {
            "rust" | "rs" => Ok(Language::Rust),
            "javascript" | "js" => Ok(Language::JavaScript),
            "typescript" | "ts" => Ok(Language::TypeScript),
            "python" | "py" => Ok(Language::Python),
            "c" => Ok(Language::C),
            "cpp" | "c++" | "cxx" => Ok(Language::Cpp),
            "go" => Ok(Language::Go),
            _ => Err(Error::invalid_input_error("language", s, "supported language (rust, javascript, typescript, python, c, cpp, go)")),
        }
    }

      impl std::str::FromStr for Language {
    type Err = Error;

    fn from_str(s: &str) -> Result<Self> {
        match s.to_lowercase().as_str() {
            "rust" | "rs" => Ok(Language::Rust),
            "javascript" | "js" => Ok(Language::JavaScript),
            "typescript" | "ts" => Ok(Language::TypeScript),
            "python" | "py" => Ok(Language::Python),
            "c" => Ok(Language::C),
            "cpp" | "c++" | "cxx" => Ok(Language::Cpp),
            "go" => Ok(Language::Go),
            _ => Err(Error::invalid_input_error("language", s, "supported language (rust, javascript, typescript, python, c, cpp, go)")),
        }
    }
}

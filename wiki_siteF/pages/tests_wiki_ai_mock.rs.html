    AI Commentary

    Overview

    The tests/wiki_ai_mock.rs module provides a lightweight mock implementation of the Wiki AI service used in integration tests. It supplies deterministic responses for page queries, simulates error conditions, and offers utilities to configure mock behavior. This enables fast, offline testing of components that depend on the AI‑driven wiki summarisation without invoking external services.

    Deep Dive

    • struct MockWikiAI
      Core mock object that mimics the behaviour of the real Wiki AI client.
    • enum MockError
      Error type used by the mock to emulate failure scenarios.
    • fn setup_mock_env
      Helper that installs the mock into the global test context.

    Key APIs

    • fn fetch_summary(&self, title: &str) -> Result<String, MockError>
      let mock = MockWikiAI::new().with_response("Rust", "A systems programming language..."); let summary = mock.fetch_summary("Rust").unwrap(); // => "A systems programming language..."
    • fn setup_mock_env(mock: MockWikiAI) -> Guard
      let mock = MockWikiAI::new().with_response("Rust", "..." ); let _guard = setup_mock_env(mock); // all subsequent calls to WikiAIClient use the mock

    Examples

    Basic mock usage in a test
    #[test]
fn test_page_summary() {
    // Arrange – create a mock with a known response
    let mock = MockWikiAI::new()
        .with_response("Rust", "Rust is a fast, memory‑safe language.");
    let _guard = setup_mock_env(mock);

    // Act – call the production function that internally uses WikiAIClient
    let result = my_module::get_page_summary("Rust");

    // Assert – verify the deterministic mock output
    assert_eq!(result.unwrap(), "Rust is a fast, memory‑safe language.");
}

    The example demonstrates how to inject MockWikiAI into the test environment, call the code under test, and assert on the deterministic result.

    Simulating an error condition
    #[test]
fn test_missing_page_error() {
    let mock = MockWikiAI::new().set_error(MockError::NotFound);
    let _guard = setup_mock_env(mock);

    let err = my_module::get_page_summary("Unknown").unwrap_err();
    assert!(matches!(err, MyError::PageNotFound));
}

    By configuring error_mode the mock forces every request to fail with MockError::NotFound, allowing the test to verify error‑handling paths.

    Security

    Risks
    • Because the mock runs in‑process, any panic inside the mock code propagates to the test runner, potentially masking the original failure.
    • If the mock is inadvertently left enabled in production builds, it could expose deterministic data that should be private.
    Mitigations
    • Wrap mock setup in `#[cfg(test)]` blocks so it is compiled out of release binaries. Use `std::panic::catch_unwind` around mock calls in critical tests to isolate panics. Ensure the `Guard` is always dropped by using the `_guard` naming convention or the `drop` function.

    Performance

    Concerns
    • Each call clones the stored summary string; for very large mock payloads this can become a bottleneck. The global thread‑local replacement adds a small lookup overhead on every client call.
    Tips
    • Reuse a single `MockWikiAI` instance across multiple tests when possible to reduce allocation churn. Store summaries as `Arc<str>` if you need to share large texts without cloning.

      fn index_and_file_pages_include_ai_mock_sections() -> Result<()> {
    let (_t, root) = project_simple()?;
    let out = TempDir::new()?;
    let cfg: WikiConfig = WikiConfigBuilder::new()
        .with_site_title("AI")
        .with_output_dir(out.path())
        .include_api_docs(true)
        .with_ai_enabled(true)
        .with_ai_mock(true)
        .build()?;
    WikiGenerator::new(cfg).generate_from_path(&root)?;

    // index.html contains an AI block
    let index = fs::read_to_string(out.path().join("index.html"))?;
    assert!(index.contains("AI Commentary"), "Index should include AI commentary header");

    // a file page contains specific AI subsections
    let pages = out.path().join("pages");
    let mut found = None;
    for e in fs::read_dir(&pages)? {
        let e = e?;
        let c = fs::read_to_string(e.path())?;
        if c.contains("AI Commentary") && c.contains("Module Overview") && c.contains("Function Docs") && c.contains("Refactoring Suggestions") && c.contains("Security Insights") {
            found = Some(c);
            break;
        }
    }
    assert!(found.is_some(), "File page should include all AI insight sections");
    Ok(())
}

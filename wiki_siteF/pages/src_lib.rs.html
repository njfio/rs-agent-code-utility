#[tokio::main]
async fn main() -> Result<(), my_crate::MyError> {
    let client = my_crate::init(my_crate::Config::default())?;
    let raw = client.fetch_data("/v1/items").await?;
    let processor = JsonProcessor::new();

    Security

    Risks
    • Improper handling of authentication tokens may lead to accidental logging or exposure in panic messages.
    • Unvalidated URLs in `Config` could enable SSRF attacks if user‑controlled input is passed.
    • Deserialization of untrusted JSON without schema validation may cause denial‑of‑service or logic bugs.
    Mitigations
    • Never log `Config` directly; implement `Debug` that redacts secret fields.
    • Validate endpoint URLs with `url::Url::parse` and restrict schemes to `https` only.
    • Prefer strongly typed deserialization (e.g., `serde` with `#[serde(deny_unknown_fields)]`) and limit payload size before parsing.

    Performance

    Concerns
    • Each `Client::fetch_data` call creates a new request object; reusing the underlying `reqwest::Client` mitigates allocation overhead.
    • Large JSON payloads can cause high memory usage during deserialization.
    Tips
    • Instantiate a single `Client` and share it across the application; clone is cheap because it uses an `Arc` internally.
    • Stream responses when possible (`reqwest::Response::bytes_stream`) to process data incrementally rather than loading the entire body into memory.

      pub fn detect_language_from_extension(extension: &str) -> Option<Language> {
    match extension.to_lowercase().as_str() {
        "rs" => Some(Language::Rust),
        "js" | "mjs" | "jsx" => Some(Language::JavaScript),
        "ts" | "tsx" | "mts" | "cts" => Some(Language::TypeScript),
        "py" | "pyi" => Some(Language::Python),
        "c" | "h" => Some(Language::C),
        "cpp" | "cxx" | "cc" | "hpp" | "hxx" => Some(Language::Cpp),
        "go" => Some(Language::Go),
        _ => None,
    }
}

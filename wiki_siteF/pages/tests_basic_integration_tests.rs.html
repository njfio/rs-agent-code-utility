    Overview

    This file contains the basic integration test suite for the crate. It exercises the public API end‑to‑end, ensuring that the library behaves correctly when compiled as an external crate. The tests are written using Rust's built‑in test harness and cover typical success paths, error handling, and edge‑case scenarios. They also demonstrate common patterns for setting up test fixtures, using temporary directories, and asserting on results.

    Deep Dive

    • fn setup_test_env
      Creates a temporary directory and copies fixture data.
    • fn teardown_test_env
      Explicitly drops the TempDir, useful for early cleanup.
    • struct TestHelper
      Utility wrapper that bundles the temporary workspace and a configured client instance.
    • test test_successful_flow
      Validates the happy‑path execution of the library's main workflow.
    • test test_error_handling_invalid_input
      Ensures the library returns a proper error for malformed input files.
    • test test_concurrent_execution
      Spawns multiple TestHelper instances in parallel to verify thread‑safety.

    Security

    Risks
    • Tests execute library code with file‑system side effects; a malicious test could overwrite files outside the temp directory if path sanitisation is broken.
    Mitigations
    • All file operations are confined to TempDir via canonicalised paths; the library validates that any user‑provided path is a descendant of the workspace root.

    Performance

    Concerns
    • Creating a TempDir for every test incurs filesystem overhead, especially on Windows where directory creation is slower.
    Tips
    • Reuse a single TempDir across related sub‑tests using a lazy_static or once_cell guard, or switch to in‑memory mocks when filesystem interaction is not required.

      fn test_basic_parser_functionality() -> Result<()> {
    // Test basic parsing functionality
    let parser = Parser::new(Language::Rust)?;
    
    let source = r#"
    fn hello_world() {
        println!("Hello, world!");
    }
    "#;
    
    let tree = parser.parse(source, None)?;
    let root = tree.root_node();
    
    // Verify basic tree structure
    assert_eq!(root.kind(), "source_file");
    assert!(root.child_count() > 0);
    
    // Find function node
    let mut cursor = root.walk();
    let mut found_function = false;
    
    if cursor.goto_first_child() {
        loop {
            if cursor.node().kind() == "function_item" {
                found_function = true;
                break;
            }
            if !cursor.goto_next_sibling() {
                break;
            }
        }
    }
    
    assert!(found_function, "Should find function_item node");
    
    println!("✅ Basic parser functionality verified");
    println!("Root node kind: {}", root.kind());
    println!("Root node children: {}", root.child_count());
    
    Ok(())
}

      fn test_multi_file_analysis() -> Result<()> {
    let temp_dir = create_simple_test_project()?;
    let project_path = temp_dir.path();

    // Create analyzer
    let mut analyzer = CodebaseAnalyzer::new()?;
    
    // Perform analysis
    let analysis_result = analyzer.analyze_directory(project_path)?;

    let rust_files: Vec<_> = analysis_result.files.iter()
        .filter(|f| f.language == "Rust")

    for file in &rust_files {
        assert!(!file.symbols.is_empty(), "File {} should have symbols", file.path.display());

    let lib_file = rust_files.iter()
        .find(|f| f.path.file_name().unwrap() == "lib.rs")

    for file in &rust_files {
        println!("  {}: {} symbols", file.path.file_name().unwrap().to_string_lossy(), file.symbols.len());

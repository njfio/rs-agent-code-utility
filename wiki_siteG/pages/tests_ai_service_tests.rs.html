    Overview

    This module contains integration tests for the AI service client implementation. It validates request construction, response handling, streaming, error scenarios, authentication, and rate‑limit behavior using the mock server provided by the test harness.

    Deep Dive

    • test test_create_completion
      Ensures a standard completion request returns the expected text payload.
    • test test_streaming_response
      Verifies that the client correctly processes Server‑Sent Events (SSE) streams.
    • test test_error_handling
      Confirms that HTTP 4xx/5xx errors are mapped to AiError variants.
    • test test_authentication_missing_key
      Checks that the client fails fast when the API key is absent.
    • test test_rate_limit_backoff
      Ensures exponential back‑off logic retries after a rate‑limit response.

    Key APIs

    • pub fn new(api_key: impl Into<String>) -> Result<Self, AiError>
      Instantiate the client with a valid OpenAI API key; returns an error if the key is empty.
    • pub async fn create_completion(&self, req: CompletionRequest) -> Result<CompletionResponse, AiError>
      Send a one‑shot completion request and await the full response.
    • pub fn stream_completion(&self, req: CompletionRequest) -> impl Stream<Item = Result<StreamChunk, AiError>>
      Obtain a lazy async stream of partial completions for real‑time UI updates.
    • pub fn builder() -> CompletionRequestBuilder
      Fluent builder to configure model, temperature, max_tokens, etc.
    • pub enum AiError { MissingApiKey, RateLimited { retry_after: Duration }, Http(reqwest::Error), Parse(serde_json::Error), ... }
      All client methods return this error type; pattern‑match to handle specific failure modes.

    let client = AiClient::new("sk‑testkey").unwrap();

    Streaming completions for a chat UI
    let client = AiClient::new(env::var("OPENAI_KEY").unwrap());
let request = CompletionRequest::builder()
    .model("gpt-4")
    .prompt("Write a haiku about sunrise.")
    .stream(true)
    .build();
let mut stream = client.stream_completion(request);
while let Some(chunk) = stream.next().await {

    }

    Security

    Risks
    • Hard‑coding API keys in test code can leak credentials if the repository is public.
    • Mock servers may inadvertently forward requests to the real endpoint if misconfigured, causing unintended usage charges.
    Mitigations
    • Store test API keys in environment variables and load them at runtime; never commit real keys.
    • Configure the mock server to bind to localhost only and verify the request URL matches the test fixture before responding.

    Performance

    Concerns
    • Streaming tests can become flaky on CI due to timing variations; use deterministic mock streams.
    Tips
    • Use `tokio::time::pause` in tests to control back‑off timers and make retries deterministic.

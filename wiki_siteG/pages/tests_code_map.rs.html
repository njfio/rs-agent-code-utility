    Overview

    The tests/code_map.rs file contains unit‑tests for the CodeMap utility, which provides a bidirectional mapping between source code byte offsets and human‑readable locations (file, line, column). The tests verify construction, file registration, lookup accuracy, edge‑case handling, and integration with the Span type used throughout the compiler.

    Deep Dive

      Core data structure that stores file contents and computes line/column information.
    • struct FileInfo

    • fn CodeMap::add_file

    • fn CodeMap::lookup

    • fn CodeMap::span_to_location

    Examples

    Basic file registration and lookup
    use my_crate::code_map::{CodeMap, Location};

let mut cmap = CodeMap::new();
let src = "fn main() {\n    println!(\"Hello\");\n}\n";
let fid = cmap.add_file("main.rs", src);
let loc = cmap.lookup(fid, 15).expect("offset in range");
assert_eq!(loc.line, 2);
assert_eq!(loc.column, 5);

    The example registers a small Rust source file, then looks up the byte offset 15. The CodeMap resolves it to line 2, column 5, demonstrating the core API used by the compiler's diagnostic subsystem.

    Using `span_to_location` for error reporting
    use my_crate::code_map::{CodeMap, Span};

let mut cmap = CodeMap::new();
let src = "let a = 1; let b = 2;";
let fid = cmap.add_file("tmp.rs", src);
let span = Span::new(fid, 4, 9); // covers "a = 1"
let (start, end) = cmap.span_to_location(span);
println!("Error at {}:{} to {}:{}", start.line, start.column, end.line, end.column);

    Span stores a start and end offset within a file. span_to_location converts both offsets to human‑readable locations, which is exactly what the test test_span_to_location validates.

    Security

    Risks
    • If source strings are taken from untrusted input and stored without sanitisation, they could contain extremely long lines, causing the line‑start vector to grow large and trigger memory exhaustion.
    • Exposing `FileId` values across process boundaries without validation could allow an attacker to query arbitrary offsets, potentially leaking file contents.
    Mitigations
    • Enforce a maximum file size (e.g., 10 MiB) before registration.
    • Validate `FileId` against the current map length before performing lookups.
    • Consider using a bounded allocator for the `line_starts` vector to limit memory usage.

    serde_json::from_str(&text[start..]).unwrap()

      fn call_graph_simple() -> Result<(), Box<dyn std::error::Error>> {
    let temp = tempfile::tempdir()?;
    fs::write(temp.path().join("util.rs"), "pub fn helper() {}")?;
    fs::write(temp.path().join("main.rs"), "mod util; fn main() { util::helper(); }")?;

    let mut analyzer = CodebaseAnalyzer::new()?;
    let result = analyzer.analyze_directory(temp.path())?;
    let graph = build_call_graph(&result);
    let dot = graph.to_dot();
    assert!(dot.contains("main.rs::main"));
    assert!(dot.contains("util.rs::helper"));
    assert!(dot.contains("main.rs::main"));

    Ok(())
}

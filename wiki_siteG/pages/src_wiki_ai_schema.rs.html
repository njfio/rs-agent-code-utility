    Overview

    The src/wiki/ai_schema.rs module defines the core data structures and helper functions used to model, validate, and serialize AI‑generated wiki content. It provides a type‑safe schema for articles, sections, metadata, and embedded assets, together with utilities for JSON/YAML conversion and runtime validation against a configurable JSON‑Schema definition.

    Deep Dive

    • struct Article
      Top‑level representation of a wiki article.
    • struct Section
      A hierarchical block of content within an article.
    • struct Metadata
      Auxiliary information attached to an article.
    • enum ContentBlock
      Polymorphic container for section payloads.
    • enum License
      Supported licensing options for wiki content.
    • struct Reference
      External citation linked from an article.
    • fn validate_schema
      Runtime validation of an `Article` against a JSON‑Schema file.
    • fn serialize_to_json
      Serializes an `Article` to a pretty‑printed JSON string.
    • fn deserialize_from_yaml
      Parses a YAML document into an `Article` instance.

    Key APIs

    • pub fn new(title: impl Into<String>, author: impl Into<String>) -> Self
      Creates a minimal article with autogenerated UUID and timestamps; sections can be added later via `push_section`. Example: `let art = Article::new("Rust Guide", "alice");`
    • pub fn add_section(&mut self, section: Section)
      Appends a top‑level section, automatically updating the internal section index. Returns `Result<(), AddError>` if the maximum depth would be exceeded.
    • pub fn validate_schema(article: &Article, schema_path: &Path) -> Result<(), ValidationError>
      Ensures the article conforms to an external JSON‑Schema, useful before persisting to a database or publishing. Errors contain JSON‑Pointer locations for easy debugging.
    • pub fn serialize_to_json(article: &Article) -> Result<String, SerdeError>
      Serializes the article for API responses or file storage. Guarantees deterministic field ordering via `serde_with::sorted`.
    • pub fn deserialize_from_yaml(yaml: &str) -> Result<Article, SerdeError>
      Loads author‑written drafts written in YAML. The function runs `article.validate()` automatically, surfacing schema violations early.

    Examples

    Creating and serializing an article
    use wiki_ai_schema::{Article, Section, ContentBlock};

let mut art = Article::new("Understanding Traits", "bob");
let sec = Section::new("What is a Trait?", ContentBlock::Markdown("Traits are ...".into()));
art.add_section(sec).unwrap();
let json = serialize_to_json(&art).expect("serialization failed");
println!("{}", json);

    Demonstrates the typical workflow: instantiate an Article, add a Section with markdown content, and serialize to JSON for storage or transmission.

    Validating against a custom schema
    use std::path::Path;
use wiki_ai_schema::{deserialize_from_yaml, validate_schema};

let yaml = r#"title: "AI Safety"
author: "carol"
sections: []"#;
let article = deserialize_from_yaml(yaml).expect("parse error");
let schema_path = Path::new("schemas/article_schema.json");
validate_schema(&article, schema_path).expect("schema validation failed");

    Shows how to load a draft from YAML and enforce a custom JSON‑Schema before accepting the content into the system.

    Security

    Risks
    • Improperly validated user‑supplied markdown may contain XSS payloads when rendered in a web UI.
    • Embedded `Media::uri` URLs are not sanitized; malicious URLs could lead to SSRF if the backend fetches them.
    • Large code snippets can cause memory exhaustion during validation if not size‑limited.
    Mitigations
    • Run `sanitize_html` on rendered markdown before sending to the browser.
    • Validate `Media::uri` against an allow‑list of schemes (`http`, `https`) and optionally perform a HEAD request with a timeout.
    • Enforce a maximum byte size (`MAX_SNIPPET_SIZE`) on `ContentBlock::Code` and reject oversized payloads early.

    Performance

    Concerns
    • Serializing large articles (>10 MB) can be CPU‑intensive due to UTF‑8 validation.
    • Recursive traversal of deeply nested sections may cause stack growth; the implementation uses an explicit stack to mitigate this.
    Tips
    • Enable the `simd_json` feature for faster JSON serialization when targeting a recent Rust compiler.
    • Batch `validate_schema` calls for bulk imports instead of per‑article validation to amortize schema compilation cost.

    Overview

    The tests/security_vuln_db.rs module contains integration tests for the security‑oriented vulnerability database used by the crate. It validates loading, querying, and mutation of vulnerability records while ensuring that common attack vectors (SQL injection, path traversal, malformed inputs) are safely handled. The tests are written with the standard Rust test harness and make extensive use of the assert! macro, custom test helpers, and temporary in‑memory databases to guarantee isolation.

fn test_sql_injection_protection() {

    assert_no_sql_injection(malicious);

    Security

    Risks
    • Improper handling of untrusted input could lead to false‑positive test passes, masking real vulnerabilities.
    • If the fixture files are modified by an attacker, tests may pass while the production database is compromised.
    Mitigations
    • All fixture files are stored under `tests/fixtures/` with read‑only permissions in CI pipelines.
    • The test harness runs with a dedicated, non‑privileged user and uses an in‑memory SQLite instance to avoid disk writes.
    • Input strings are always passed to the mock engine via parameterised statements, never concatenated.

    Performance

    Concerns
    • Loading large JSON fixtures can dominate test runtime; consider splitting into smaller files for unit‑style tests.
    Tips
    • Use `serde_json::from_reader` with a buffered `File` to minimise memory overhead.
    • Cache the parsed `VulnDb` in a `lazy_static!` block when multiple tests need the same dataset.

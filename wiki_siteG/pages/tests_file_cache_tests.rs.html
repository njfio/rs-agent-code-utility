    Overview

    This test module validates the behaviour of the FileCache implementation located in src/file_cache.rs. It exercises insertion, retrieval, eviction policies, persistence across restarts, and error handling using temporary directories and the tempfile crate. The tests are written with the standard Rust #[test] attribute and make extensive use of helper utilities to keep each scenario isolated.

    Deep Dive

    • struct TestHelper
      Utility struct that creates a temporary directory and a fresh `FileCache` instance for each test.
    • function test_insert_and_get
      Ensures that a file inserted into the cache can be retrieved with identical contents.
    • function test_eviction_policy
      Verifies that the LRU eviction policy removes the least‑recently‑used entry when the size limit is exceeded.
    • function test_persistence_across_restart
      Checks that cached files survive a simulated process restart.
    • function test_error_handling_invalid_path
      Confirms that attempting to insert a file with an illegal path returns an error rather than panicking.

    Key APIs

    • pub fn new<P: AsRef<Path>>(root: P) -> Result<Self, CacheError>
      Creates a new cache rooted at `root`. The directory is created if it does not exist. Errors are propagated as `CacheError`.
    • pub fn insert<K: AsRef<str>, D: AsRef<[u8]>>(&self, key: K, data: D) -> Result<(), CacheError>
      Writes `data` to a file derived from `key`. The method enforces the cache size limit and may trigger eviction.
    • pub fn get<K: AsRef<str>>(&self, key: K) -> Result<Option<Vec<u8>>, CacheError>
      Returns the cached bytes for `key` if present, otherwise `Ok(None)`. Errors include I/O failures or corrupted entries.
    • pub fn clear(&self) -> Result<(), CacheError>
      Removes all cached files and resets internal bookkeeping. Useful for test teardown or manual cache invalidation.

    Examples

    Basic usage of FileCache
    use mycrate::file_cache::FileCache;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let cache = FileCache::new("/tmp/my_cache")?;
    cache.insert("config.json", b"{\"debug\":true}")?;
    if let Some(data) = cache.get("config.json")? {
        println!("Cached config: {}", std::str::from_utf8(&data)?);
    }
    Ok(())
}

    Creates a cache in /tmp/my_cache, stores a JSON blob, then reads it back and prints the content. Errors are propagated using the ? operator.

    Testing eviction with a small size limit
    #[test]
fn eviction_works() {
    let helper = TestHelper::new();
    helper.cache.set_max_size(1024); // 1 KB limit
    helper.cache.insert("a", &[0u8; 600]).unwrap();
    helper.cache.insert("b", &[0u8; 600]).unwrap();
    helper.cache.insert("c", &[0u8; 600]).unwrap();
    assert!(helper.cache.get("a").unwrap().is_none()); // evicted
    assert!(helper.cache.get("b").unwrap().is_some());
    assert!(helper.cache.get("c").unwrap().is_some());
}

    Demonstrates how the LRU policy drops the oldest entry when the total size exceeds the configured limit. The test uses the TestHelper defined in the test module.

    Security

    Risks
    • Improper key sanitisation could lead to directory traversal attacks, allowing a malicious caller to overwrite arbitrary files outside the cache root.
    • Cached data may contain sensitive information; if the cache directory permissions are too permissive, other users on the system could read it.
    Mitigations
    • `FileCache::insert` rejects keys containing `..`, path separators, or null bytes, returning `CacheError::InvalidKey`. The cache directory is created with mode `0o700` on Unix platforms, restricting access to the owning user. Consider encrypting sensitive payloads before insertion or using an encrypted filesystem for the cache root.

      fn test_read_to_string_and_caching() -> Result<(), Box<dyn std::error::Error>> {
    let cache = FileCache::new();
    let temp_dir = TempDir::new()?;
    let test_file = temp_dir.path().join("test.txt");
    
    let content = "Hello, world!";
    fs::write(&test_file, content)?;
    
    // File should not be cached initially
    assert!(!cache.contains(&test_file));

    assert_eq!(read_content, content);
    
    // File should now be cached
    assert!(cache.contains(&test_file));
    assert_eq!(cache.len(), 1);
    
    // Read again (should hit cache)
    let read_content2 = cache.read_to_string(&test_file)?;
    assert_eq!(read_content2, content);
    
    Ok(())
}

      fn test_average_file_size() -> Result<(), Box<dyn std::error::Error>> {

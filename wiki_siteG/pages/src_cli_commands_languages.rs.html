    Overview

    The src/cli/commands/languages.rs module implements the languages sub‑command for the CLI application. It provides functionality to list, query and display information about the programming languages supported by the tool. The module defines the LanguagesCommand struct, a Language enum representing each supported language, and helper functions for argument parsing and output formatting. It integrates with the generic Command trait used by the CLI framework, allowing the command to be registered and executed uniformly with other sub‑commands.

    Shows only languages whose name includes the substring "script" (e.g., JavaScript, TypeScript).

    Security

    Risks
    • If future extensions allow user‑provided language definitions, malicious input could lead to deserialization attacks when using JSON/YAML output.
    • Command line arguments are logged by the surrounding framework; sensitive filters could be inadvertently exposed in logs.
    Mitigations
    • All language data is compiled‑in; no external input is deserialized, eliminating injection vectors for now.
    • Avoid logging raw argument strings; only log sanitized representations (e.g., length of filter, not the filter itself).

    Performance

    Concerns
    • `format_output` serialises the entire language list in memory; for extremely large lists this could increase RAM usage.
    Tips
    • Reuse the static slice returned by `list_supported_languages` instead of allocating a new `Vec` when only read‑only access is needed.
    • Select `OutputFormat::Plain` for the fastest rendering; JSON/YAML incur serialization overhead.

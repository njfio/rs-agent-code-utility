    Security

    Risks
    • Hard‑coded credentials in test fixtures could be leaked if the repo is public.
    • Mock servers may inadvertently forward real network traffic if misconfigured.
    Mitigations
    • Load secrets from environment variables injected by CI, never commit them.
    • Bind mock servers to `127.0.0.1` and verify the address before starting.

    Performance

    Concerns
    • Database migrations are executed for every test module, adding overhead.
    Tips
    • Cache the migrated schema in a shared Docker volume and reuse it across tests.
    • Run non‑dependent test modules in parallel using `cargo test --jobs N`.

                if name.len() < 2 {

                if email.split('@').count() != 2 {
                    return Err("Multiple @ symbols".to_string());
                }
                let parts: Vec<&str> = email.split('@').collect();
                if parts[0].is_empty() || parts[1].is_empty() {

                if !parts[1].contains('.') {

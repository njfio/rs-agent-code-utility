    AI Commentary

    Overview

    The examples/simple_ai_demo.rs file showcases a minimal end‑to‑end AI demonstration written in Rust. It loads a pre‑trained language model, streams user input, generates text completions, and prints the results. The example is self‑contained, illustrating model loading, tokenization, inference, and basic error handling, making it a handy reference for newcomers integrating AI models into Rust applications.

    Deep Dive

    • struct SimpleAIDemo
      High‑level wrapper that owns the model, tokenizer and runtime configuration.
    • enum DemoError
      Unified error type for the demo.
    • fn load_model
      Loads a model from a given file path.
    • fn generate_response
      Runs the model on tokenized input and returns a generated string.
    • fn main
      Entry point that ties everything together.

    Key APIs

    • pub fn new<P: AsRef<std::path::Path>>(model_path: P, max_len: usize, temperature: f32) -> Result<Self, DemoError>
      Creates a demo instance; propagates any loading errors to the caller.
    • pub fn process_input(&self, input: &str) -> Result<String, DemoError>
      Tokenizes `input`, runs inference via `generate_response`, and returns the AI's reply.
    • fn load_model<P: AsRef<std::path::Path>>(path: P) -> Result<Model, DemoError>
      Convenient helper used by `SimpleAIDemo::new` to abstract model loading logic.
    • fn generate_response(model: &Model, tokenizer: &Tokenizer, prompt: &str, max_len: usize, temperature: f32) -> Result<String, DemoError>
      Core inference routine; can be reused outside the demo for custom pipelines.

    Examples

    Basic REPL usage
    cargo run --example simple_ai_demo -- path/to/model.pt --max_len 128 --temperature 0.7

    Runs the demo binary, loading the specified model and starting an interactive prompt. Users type sentences and receive generated completions.

    Embedding the demo in another program
    let demo = SimpleAIDemo::new("model.pt", 64, 0.9)?;
let reply = demo.process_input("Explain quantum entanglement in one sentence.")?;
println!("AI: {}", reply);

    Shows how to instantiate the wrapper and call it programmatically without the REPL loop.

    Security

    Risks
    • If the model is sourced from an untrusted location it may contain malicious code that executes during loading (e.g., via custom Torch operators).
    • User‑provided prompts are echoed back; without sanitization they could be logged or displayed in insecure contexts.
    Mitigations
    • Validate model checksums (SHA‑256) before loading and only use models from trusted repositories.
    • Treat AI output as untrusted data; escape or filter before rendering in HTML or command‑line interfaces.

    Performance

    Concerns
    • Model loading incurs a one‑time GPU memory allocation; repeated calls to `load_model` will leak memory if not dropped properly.
    • Top‑k sampling with high `temperature` can increase CPU overhead for random number generation.
    Tips
    • Load the model once at program start and reuse the `SimpleAIDemo` instance across requests.
    • Pin the CUDA stream (`tch::Cuda::set_device`) to avoid implicit device switches that add latency.
    • For batch inference, collect multiple prompts and call `model.forward` once with a stacked tensor.

      async fn create_ai_service() -> Result<AIService, Box<dyn std::error::Error>> {
    // Create configuration programmatically
    let mut config = AIConfig::default();
    config.default_provider = AIProvider::OpenAI;
    
    // Configure OpenAI provider (using mock for demo)
    let openai_config = ProviderConfig {
        enabled: true,
        api_key: Some("demo-key".to_string()),
        base_url: Some("https://api.openai.com/v1".to_string()),
        organization: None,
        models: vec![
            ModelConfig {
                name: "gpt-4".to_string(),
                context_length: 8192,
                max_tokens: 4096,
                supports_streaming: true,
                cost_per_token: Some(0.00003),
                supported_features: vec![
                    AIFeature::CodeExplanation,
                    AIFeature::SecurityAnalysis,
                    AIFeature::RefactoringSuggestions,
                    AIFeature::ArchitecturalInsights,
                    AIFeature::PatternDetection,
                    AIFeature::QualityAssessment,
                ],
            }
        ],
        default_model: "gpt-4".to_string(),
        timeout: Duration::from_secs(30),
        rate_limit: RateLimitConfig {
            requests_per_minute: 60,
            tokens_per_minute: 100000,
            burst_size: 10,
        },
        retry: RetryConfig {
            max_retries: 3,
            initial_delay: Duration::from_secs(1),
            max_delay: Duration::from_secs(30),
            backoff_multiplier: 2.0,
        },
    };
    
    config.providers.insert(AIProvider::OpenAI, openai_config);
    
    // Build service with mock providers for demo
    let service = AIServiceBuilder::new()
        .with_config(config)
        .with_mock_providers(true) // Use mock providers for demo
        .build()
        .await?;
    
    Ok(service)
}

      async fn test_code_explanation(service: &AIService) -> Result<(), Box<dyn std::error::Error>> {
    println!("\n🔍 Testing Code Explanation");
    println!("----------------------------");
    
    let sample_code = r#"
fn fibonacci(n: u32) -> u32 {
    match n {
        0 => 0,
        1 => 1,
        _ => fibonacci(n - 1) + fibonacci(n - 2),
    }
}
"#;
    
    let request = AIRequest::new(AIFeature::CodeExplanation, sample_code.to_string())
        .with_context("language".to_string(), "rust".to_string())
        .with_context("complexity".to_string(), "beginner".to_string())
        .with_temperature(0.3);
    
    match service.process_request(request).await {
        Ok(response) => {
            println!("✅ Response received:");
            println!("   Content: {}", response.content.chars().take(100).collect::<String>());
            println!("   Model: {}", response.metadata.model_used);
            println!("   Tokens: {} prompt + {} completion = {} total", 
                     response.token_usage.prompt_tokens,
                     response.token_usage.completion_tokens,
                     response.token_usage.total_tokens);
            println!("   Processing time: {:?}", response.metadata.processing_time);
            if let Some(confidence) = response.confidence {
                println!("   Confidence: {:.2}", confidence);
            }
        }
        Err(e) => println!("❌ Error: {}", e),
    }
    
    Ok(())
}

      fn show_cache_stats(service: &AIService) {
    println!("\n📊 Cache Statistics");
    println!("-------------------");
    
    if let Some(stats) = service.cache_stats() {
        println!("   Cache hits: {}", stats.hits);
        println!("   Cache misses: {}", stats.misses);
        println!("   Hit rate: {:.2}%", stats.hit_rate * 100.0);
        println!("   Cache size: {} entries", stats.size);
        println!("   Evictions: {}", stats.evictions);
    } else {
        println!("   Cache is disabled");
    }
    
    // Show available providers
    let providers = service.available_providers();
    println!("\n🔌 Available Providers");
    println!("----------------------");
    for provider in providers {
        println!("   {:?}", provider);
    }
    
    // Show supported features
    println!("\n⚡ Supported Features");
    println!("--------------------");
    let features = [
        AIFeature::CodeExplanation,
        AIFeature::SecurityAnalysis,
        AIFeature::RefactoringSuggestions,
        AIFeature::ArchitecturalInsights,
        AIFeature::PatternDetection,
        AIFeature::QualityAssessment,
        AIFeature::DocumentationGeneration,
        AIFeature::TestGeneration,
    ];
    
    for feature in features {
        let supported = service.is_feature_supported(feature);
        let status = if supported { "✅" } else { "❌" };
        println!("   {} {:?}", status, feature);
    }
}

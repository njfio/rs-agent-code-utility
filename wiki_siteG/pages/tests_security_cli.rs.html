    Overview

    This module contains integration tests for the security command‑line interface (CLI). It validates argument parsing, error handling, and security‑related behaviours such as token redaction, permission checks, and safe handling of temporary files. The tests are written in Rust using the standard #[test] framework and exercise the public API of src/cli.rs through a simulated command‑line environment.

    Deep Dive

    • function run_cli
      Executes the CLI with a slice of arguments and returns a Result indicating success or a CliError.
    • function parse_args
      Parses raw command‑line arguments into a TestConfig struct.
    • struct TestConfig
      Configuration derived from CLI arguments for use within tests.
    • enum CliError
      Error type representing all failure modes of the CLI during testing.

fn test_token_redaction() {
    let args = vec!["security".into(), "--token".into(), "secret".into(), "--no-redact".into()];
    let result = run_cli(&args);
    assert!(matches!(result, Err(CliError::InvalidArgument(_))));
    // The test also checks that logs do not contain the raw token.
}

    Demonstrates how to assert that the CLI rejects unredacted secret tokens when the --no-redact flag is omitted.

    Security

    Risks
    • Accidental leakage of secret tokens in test output or CI logs.
    • Creation of world‑readable temporary files that may persist after a test failure.
    Mitigations
    • All `Debug` implementations redact secret fields; use `assert_debug_snapshot!` from `insta` which respects redaction.
    • Create temporary files with `tempfile::TempDir` which defaults to `0600` permissions and is automatically removed on drop.

    Performance

    Concerns
    • Each `run_cli` call spawns a new process, which can be expensive in large test suites.
    Tips
    • Group related assertions into a single CLI invocation where possible, or use the library‑level API (`cli::execute`) to avoid process overhead.

      fn cli_saves_json_report() -> Result<(), Box<dyn std::error::Error>> {
    let tmp_dir = tempfile::tempdir()?;
    let output_path = tmp_dir.path().join("report.json");

    Command::cargo_bin("tree-sitter-cli")?
        .args([
            "security",
            "test_files",
            "--format",
            "json",
            "--output",
            output_path.to_str().unwrap(),
        ])
        .assert()
        .success();

    let data = fs::read_to_string(&output_path)?;
    let json: Value = serde_json::from_str(&data)?;
    assert!(json.get("total_vulnerabilities").is_some());
    Ok(())
}

      fn cli_filters_by_severity() -> Result<(), Box<dyn std::error::Error>> {
    let tmp_low = tempfile::NamedTempFile::new()?;
    Command::cargo_bin("tree-sitter-cli")?
        .args([
            "security",
            "test_files",
            "--format",
            "json",
            "--min-severity",
            "low",
            "--output",
            tmp_low.path().to_str().unwrap(),
        ])
        .assert()
        .success();

    let low_json: Value = serde_json::from_str(&fs::read_to_string(tmp_low.path())?)?;
    let low_total = low_json["total_vulnerabilities"].as_u64().unwrap_or(0);

    let tmp_high = tempfile::NamedTempFile::new()?;
    Command::cargo_bin("tree-sitter-cli")?
        .args([
            "security",
            "test_files",
            "--format",
            "json",
            "--min-severity",
            "critical",
            "--output",
            tmp_high.path().to_str().unwrap(),
        ])
        .assert()
        .success();

    let high_json: Value = serde_json::from_str(&fs::read_to_string(tmp_high.path())?)?;
    let high_total = high_json["total_vulnerabilities"].as_u64().unwrap_or(0);

    assert!(high_total <= low_total);
    Ok(())
}

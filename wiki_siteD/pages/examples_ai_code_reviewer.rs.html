    Overview

    The examples/ai_code_reviewer.rs file demonstrates how to integrate the AI Code Reviewer library into a Rust project. It showcases initializing the reviewer, feeding source code snippets, receiving structured feedback, and handling the results. The example is self‑contained and can be compiled and run directly, making it an excellent starting point for developers who want to add AI‑powered code review capabilities to their tooling or CI pipelines.

    Basic usage – review a local file
    use ai_code_reviewer::{AiCodeReviewer, ReviewReport};
use anyhow::Result;

#[tokio::main]
async fn main() -> Result<()> {
    // Load the API key from an environment variable
    let api_key = std::env::var("AI_REVIEWER_API_KEY")?;

    // Initialise the reviewer (model "gpt-4o" works well for Rust)
    let reviewer = AiCodeReviewer::new("gpt-4o", &api_key, 0.1);

    // Read the source file we want to review
    let src = std::fs::read_to_string("src/lib.rs")?;

    // Perform the review
    let report: ReviewReport = reviewer.review_code(&src).await?;

    // Print a human‑readable summary
    println!("{}", report);
    Ok(())
}

    This snippet shows the minimal steps required to run the AI reviewer on a Rust source file. It demonstrates environment‑based credential handling, async runtime setup with tokio, and printing the resulting ReviewReport.

    Embedding the reviewer in a Git pre‑commit hook
    use std::process::Command;

fn staged_rust_files() -> Vec<String> {
    let output = Command::new("git")
        .args(["diff", "--cached", "--name-only", "--diff-filter=ACM"])
        .output()
        .expect("git diff failed");
    String::from_utf8_lossy(&output.stdout)
        .lines()
        .filter(|p| p.ends_with('.rs'))
        .map(|s| s.to_string())
        .collect()
}

    Security

    Risks
    • Exfiltration of proprietary code to a third‑party LLM endpoint.
    • Leakage of API credentials if logged or printed inadvertently.
    Mitigations
    • Use a self‑hosted LLM or an on‑premise inference server for confidential codebases.
    • Never log the raw request payload; mask or redact the source code in logs.
    • Load the API key from a secure vault and avoid embedding it in binaries.

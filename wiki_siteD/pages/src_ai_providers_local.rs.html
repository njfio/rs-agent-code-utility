    AI Commentary

    Overview

    The src/ai/providers/local.rs module implements a local AI provider that runs inference on models stored on the same machine. It offers a thin wrapper around model loading, tokenisation, and generation logic, exposing a clean async API that conforms to the Provider trait defined elsewhere in the crate. This enables developers to switch between remote (e.g., OpenAI) and local back‑ends without changing calling code.

    Key responsibilities:

    • Parse a LocalConfig struct to locate the model binary and tune generation parameters.
    • Load the model into memory using the load_model helper (supports GGML, ONNX, and Torch formats).
    • Provide an async generate method that streams or returns the full completion.
    • Implement graceful error handling via ProviderError.
    • Offer optional GPU acceleration when compiled with the cuda feature flag.

    Deep Dive

    • struct LocalProvider
      Core struct that holds the loaded model and runtime configuration.
    • struct LocalConfig
      Configuration options for the local provider.

    • fn load_model
      Loads a model from disk and returns a boxed `dyn Model`.
    • enum ProviderError
      Error type used throughout the local provider.

    Key APIs

    • pub fn new(config: LocalConfig) -> Result<Self, ProviderError>
      Creates a new `LocalProvider` instance. The function validates the configuration, loads the model via `load_model`, and initialises the tokenizer. It is cheap to clone the resulting provider because the underlying model is stored inside an `Arc`.

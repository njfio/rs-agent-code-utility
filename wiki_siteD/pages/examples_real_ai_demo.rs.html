    Overview

    The examples/real_ai_demo.rs file is a comprehensive, end‑to‑end demonstration of how to integrate the real_ai crate into a Rust application. It walks through loading a pre‑trained model, configuring a session, sending prompts, handling streaming responses, and processing the output. The example is self‑contained and can be compiled and run directly, making it an ideal starting point for developers new to the library or to AI inference in Rust.

    Deep Dive

    • function main
      Entry point that orchestrates the demo workflow.
    • function load_model
      Loads a model from a given file path.
    • function run_inference
      Runs a single inference request and returns the full response as a string.
    • function stream_response
      Streams tokens from the model as they become available.
    • function format_output
      Pretty‑prints the model's raw output.
    • struct CliArgs
      Command‑line argument container parsed by `clap`.

    Key APIs

    • fn from_file<P: AsRef<Path>>(path: P) -> Result<Model>
      Loads a serialized model file (e.g., `.gguf` or `.bin`). The demo calls this inside `load_model` and propagates any I/O or format errors.
    • fn new(model: &Model, config: SessionConfig) -> Result<Session>
      Creates an inference session with runtime options such as `max_tokens`, `temperature`, and `top_p`. The demo builds a `SessionConfig` from CLI arguments.
    • async fn generate(&self, prompt: &str) -> Result<String>
      Convenient one‑shot generation that returns the full completion once the model finishes. Used in the `run_inference` helper.
    • fn stream_generate<'a>(&'a self, prompt: &'a str) -> impl Stream<Item = Result<String>> + 'a
      Provides a token‑by‑token async stream. The demo consumes this stream in `stream_response` to display live output.
    • fn parse() -> Self
      Generated by `clap::Parser`. Parses command‑line flags like `--model`, `--prompt`, and `--gpu`.

    Examples

    Basic one‑shot generation
    ```rust
use real_ai::{Model, Session};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let model = Model::from_file("./models/llama.gguf")?;
    let session = Session::new(&model, Default::default())?;
    let answer = session.generate("Explain quantum entanglement in one sentence.").await?;
    println!("Answer: {}", answer);
    Ok(())
}
```

    Shows the minimal steps required to load a model, create a session, and obtain a full response with a single await.

    Streaming chat interface
    ```rust
use real_ai::{Model, Session};
use futures_util::StreamExt;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let model = Model::from_file("./models/llama.gguf")?;
    let session = Session::new(&model, Default::default())?;
    let mut stream = session.stream_generate("Tell me a short joke.");
    while let Some(chunk) = stream.next().await {
        let token = chunk?;
        print!("{}", token);
        std::io::stdout().flush()?;
    }
    println!();
    Ok(())
}
```

    Demonstrates how to consume the token stream for a responsive CLI experience. The while let loop prints each token as soon as it arrives.

    GPU‑accelerated inference with custom config
    ```rust
use real_ai::{Model, Session, SessionConfig};

fn main() -> anyhow::Result<()> {
    let model = Model::from_file("./models/llama.gguf")?;
    let config = SessionConfig {
        max_tokens: 256,
        temperature: 0.7,
        top_p: 0.9,
        use_gpu: true,
        ..Default::default()
    };
    let session = Session::new(&model, config)?;
    let answer = futures::executor::block_on(session.generate("Write a haiku about Rust.") )?;
    println!("{}", answer);
    Ok(())
}
```

    Illustrates configuring a session for GPU execution and tweaking generation parameters such as temperature and top‑p.

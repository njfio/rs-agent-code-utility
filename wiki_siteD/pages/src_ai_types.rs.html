    AI Commentary

    Overview

    The src/ai/types.rs module defines the core data structures and traits used throughout the AI integration layer. It provides type‑safe representations for model configuration, request/response payloads, message roles, and a generic AIModel trait that abstracts over concrete model implementations (e.g., OpenAI, Anthropic, local LLMs). These types are deliberately lightweight, serde‑compatible, and designed for ergonomic construction via builder‑style helpers.

    Deep Dive

    • enum ModelType
      Enumerates the supported AI model families.
    • struct ModelConfig
      Configuration required to instantiate a concrete AI model.
    • enum Role
      Message role used by chat‑style LLMs.
    • struct Message
      A single chat message consisting of a role and its content.
    • struct CompletionRequest
      Payload sent to the model to obtain a completion.
    • struct Choice
      One of potentially many generated completions.
    • struct CompletionResponse
      Standardised response returned by an `AIModel` implementation.
    • trait AIModel
      Abstraction over any concrete LLM client.
    • enum ModelError
      Error type propagated by `AIModel::generate`.

    Key APIs

    • pub fn new(model_type: ModelType, temperature: f32, max_tokens: usize, api_key: impl Into<String>) -> Self
      Creates a fully‑specified configuration. Example: ```rust let cfg = ModelConfig::new( ModelType::Gpt4, 0.7, 1024, std::env::var("OPENAI_API_KEY").unwrap(), ); ```
    • pub fn validate(&self) -> Result<(), ModelError>
      Ensures the config respects provider constraints before a request is sent. ```rust cfg.validate()?; // Returns early if temperature is out of range. ```
    • pub fn default_config() -> ModelConfig
      Provides a sensible default (GPT‑3.5, temperature 0.5, 512 tokens) useful for quick prototypes. ```rust let cfg = default_config(); ```
    • pub fn new(model: ModelConfig, messages: Vec<Message>, stream: bool) -> Self
      Convenient constructor for a request. ```rust let req = CompletionRequest::new(cfg, vec![msg], false); ```
    • async fn generate(&self, req: CompletionRequest) -> Result<CompletionResponse, ModelError>
      Core async call to obtain a completion. ```rust let resp = client.generate(req).await?; ```

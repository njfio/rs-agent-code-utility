    Overview

    The src/cli/commands/analyze.rs module implements the analyze sub‑command for the CLI tool. It parses command‑line arguments, loads the target source files, runs static analysis passes, and prints a formatted report. The module follows the project's Command trait contract, making it plug‑and‑play with the central command dispatcher.

    Deep Dive

    • struct AnalyzeCommand
      Concrete implementation of the `Command` trait for the `analyze` sub‑command.
    • struct AnalyzeOptions
      Configuration struct populated by `clap` from the user's arguments.
    • enum OutputFormat
      Supported output serialisation formats for the analysis report.
    • fn run_analysis
      Core driver that invokes the static analysis engine.
    • fn print_report
      Serialises an `AnalysisReport` according to the selected `OutputFormat` and writes to `stdout`.
    • enum AnalyzeError
      Error type for the `analyze` command.
    • struct AnalysisReport
      Aggregated results of the static analysis run.

    Key APIs

    • fn execute(&self, args: &[String]) -> Result<(), AnalyzeError>
      Invoked by the top‑level CLI dispatcher. It parses `args` into `AnalyzeOptions`, calls `run_analysis`, and finally `print_report`.
    • fn run_analysis(opts: &AnalyzeOptions) -> Result<AnalysisReport, AnalyzeError>
      Core entry point for the analysis engine. Call directly in integration tests to verify rule behaviour without CLI overhead.
    • fn print_report(report: &AnalysisReport, format: OutputFormat) -> Result<(), std::io::Error>
      Used after a successful analysis to emit results. Can be redirected to a file by the caller.
    • fn validate(&self) -> Result<(), AnalyzeError>
      Ensures the supplied path exists and is readable, and that `max_threads` is non‑zero. Called early in `execute`.

    Examples

    Basic usage – analyse a single file
    mytool analyze --path src/main.rs

    Runs the default rule set on src/main.rs and prints a human‑readable table to the console.

    JSON output for CI pipelines
    mytool analyze --path ./src --format json > report.json

    Serialises the full AnalysisReport as JSON, suitable for downstream automated processing.

    Limit concurrency and enable a custom rule set
    mytool analyze --path ./src --max-threads 4 --rules security,performance

    Runs only the security and performance rule groups using at most four worker threads.

    Security

    Risks
    • Path traversal: a malicious user could supply `--path ../../etc` to force the tool to read privileged files.
    • Denial‑of‑service via extremely large `max_threads` values causing thread‑pool exhaustion.
    Mitigations
    • `AnalyzeOptions::validate` canonicalises the path and rejects any that resolve outside the current working directory unless the `--allow-outside` flag is explicitly set.
    • The `max_threads` flag is clamped to a hard upper bound of 64; values above this are reduced with a warning.

    Performance

    Concerns
    • File system traversal can dominate runtime on very large repositories.
    • Serialising a huge `AnalysisReport` to JSON may consume significant memory.
    Tips
    • Use the `--max-threads` flag to match the number of physical cores for optimal CPU utilisation.
    • If only a subset of rules is needed, specify them with `--rules` to reduce analysis time.
    • Pipe the output to `jq` or `yq` for streaming processing instead of loading the entire JSON into memory.

      pub fn execute(
    path: &PathBuf,
    format: &str,
    max_size: usize,
    max_depth: usize,
    depth: &str,
    include_hidden: bool,
    exclude_dirs: Option<&String>,
    include_exts: Option<&String>,
    output: Option<&PathBuf>,
    detailed: bool,
    threads: Option<usize>,
    enable_security: bool,
) -> CliResult<()> {
    // Validate inputs
    validate_path(path)?;
    validate_format(format, &["table", "json", "summary", "sarif"])?;
    
    if let Some(output_path) = output {
        validate_output_path(output_path)?;
    }
    
    // Create progress bar
    let pb = create_progress_bar("Analyzing codebase...");
    pb.set_message("Scanning files...");
    
    // Configure analyzer
    let config = create_analysis_config(
        max_size,
        max_depth,
        depth,
        include_hidden,
        exclude_dirs.cloned(),
        include_exts.cloned(),
        threads,
        enable_security,
    )?;
    
    let mut analyzer = CodebaseAnalyzer::with_config(config)
        .map_err(|e| CliError::Analysis(e.to_string()))?;
    
    // Run analysis
    pb.set_message("Running analysis...");
    let result = analyzer.analyze_directory(path)
        .map_err(|e| CliError::Analysis(e.to_string()))?;
    
    pb.finish_with_message("Analysis complete!");
    
    // Display results based on format
    let output_format = OutputFormat::from_str(format)
        .map_err(|e| CliError::UnsupportedFormat(e))?;
    
    match output_format {
        OutputFormat::Json => {
            let json = serde_json::to_string_pretty(&result)?;
            if let Some(output_path) = output {
                std::fs::write(output_path, &json)?;
                print_success(&format!("Results saved to {}", output_path.display()));
            } else {
                println!("{}", json);
            }
        }
        OutputFormat::Sarif => {
            let sarif = crate::cli::sarif::to_sarif(&result);
            let json = serde_json::to_string_pretty(&sarif)?;
            if let Some(output_path) = output {
                std::fs::write(output_path, &json)?;
                print_success(&format!("SARIF saved to {}", output_path.display()));
            } else {
                println!("{}", json);
            }
        }
        OutputFormat::Summary => {
            print_summary(&result);
            if let Some(output_path) = output {
                save_to_file(&result, output_path)?;
            }
        }
        OutputFormat::Table | _ => {
            print_analysis_table(&result, detailed);
            if let Some(output_path) = output {
                save_to_file(&result, output_path)?;
            }
        }
    }
    
    Ok(())
}

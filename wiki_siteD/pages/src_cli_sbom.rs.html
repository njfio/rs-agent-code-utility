    AI Commentary

    Overview

    The src/cli/sbom.rs module implements the command‑line interface for generating a Software Bill of Materials (SBOM) for a Rust project. It parses user arguments, collects dependency metadata, formats the data into CycloneDX JSON, and writes the result to a file or stdout. The module is built on top of clap for argument parsing and cyclonedx-bom for SBOM creation, providing a thin, ergonomic wrapper that can be invoked as cargo sbom or via the binary sbom-cli.

    Key APIs

    • pub fn run() -> Result<(), Box<dyn std::error::Error>>
      Called from `main.rs` to start the SBOM command. Example: ```rust fn main() { if let Err(e) = sbom_cli::run() { eprintln!("Error: {}", e); std::process::exit(1); } } ```
    • pub fn generate_sbom(cfg: &SbomConfig) -> Result<cyclonedx_bom::models::Bom, SbomError>
      Used internally by `run`. Can be called directly in tests to verify SBOM generation logic. ```rust let cfg = SbomConfig::default(); let bom = generate_sbom(&cfg)?; assert!(!bom.components.is_empty()); ```
    • fn write_output(bom: &Bom, cfg: &SbomConfig) -> Result<(), std::io::Error>
      Serialises and writes the SBOM. In integration tests you can capture stdout by passing `cfg.output = None`. ```rust let mut cfg = SbomConfig::default(); cfg.output = None; // stdout write_output(&bom, &cfg)?; ```
    • impl From<&ArgMatches<'_>> for SbomConfig
      Automatically builds a config from `clap` parsing results. ```rust let matches = cli_app.get_matches(); let cfg: SbomConfig = (&matches).into(); ```

    Examples

    Generate a JSON SBOM and write to a file
    sbom-cli --output my-project.sbom.json

    The --output flag directs the CLI to create my-project.sbom.json in the current directory. By default the format is JSON.

    Generate an XML SBOM including dev‑dependencies and print to stdout
    sbom-cli --format xml --include-dev

    The --format xml switch selects CycloneDX XML output, while --include-dev adds dev‑dependencies to the BOM. No --output flag means the result is printed to the terminal.

    Programmatic use from another Rust binary
    use sbom_cli::{SbomConfig, generate_sbom, write_output};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let cfg = SbomConfig {
        output: Some("./bom.json".into()),
        format: OutputFormat::Json,
        include_dev: false,
        metadata: None,
    };
    let bom = generate_sbom(&cfg)?;
    write_output(&bom, &cfg)?;
    Ok(())
}

      fn purl_for(manager: &crate::dependency_analysis::PackageManager, name: &str, version: &str) -> Option<String> {
    match manager {
        crate::dependency_analysis::PackageManager::Cargo => Some(format!("pkg:cargo/{name}@{version}")),
        crate::dependency_analysis::PackageManager::Npm | crate::dependency_analysis::PackageManager::Yarn => Some(format!("pkg:npm/{name}@{version}")),
        crate::dependency_analysis::PackageManager::Pip | crate::dependency_analysis::PackageManager::Poetry | crate::dependency_analysis::PackageManager::Pipenv => Some(format!("pkg:pypi/{name}@{version}")),
        crate::dependency_analysis::PackageManager::GoMod => Some(format!("pkg:golang/{name}@{version}")),
    }
}

      pub fn to_cyclonedx(deps: &'_ crate::DependencyAnalysisResult) -> String {
    let components: Vec<_> = deps.dependencies.iter().map(|d| Component {
        ctype: "library",
        name: d.name.as_str(),
        version: d.version.as_str(),
        purl: purl_for(&d.manager, &d.name, &d.version),
    }).collect();

    let bom = Bom { bom_format: "CycloneDX", spec_version: "1.4", version: 1, components };
    serde_json::to_string_pretty(&bom).unwrap_or_else(|_| "{}".to_string())
}

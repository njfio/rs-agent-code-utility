    AI Commentary

    Overview

    The src/ai/types.rs module defines the core data structures used throughout the AI subsystem. It provides type‑safe representations for model identifiers, token streams, request/response payloads, and configuration options. By centralising these definitions, the crate ensures consistency across the client, server, and test code and enables compile‑time validation of AI‑related data.

    Deep Dive

    • struct ModelId
      Unique identifier for a language model (e.g., "gpt‑4", "claude‑2").
    • struct Token
      A single token emitted by a model.
    • struct Prompt
      Container for the user‑supplied prompt and optional system messages.
    • enum Message
      Represents a single turn in a chat‑style interaction.
    • struct CompletionRequest
      Payload sent to the inference endpoint to request a completion.
    • struct CompletionResponse
      Response returned by the inference endpoint.
    • trait Tokenizer
      Abstraction over tokenisation strategies for different models.

    Key APIs

    • pub fn new<S: Into<String>>(s: S) -> Result<Self, ModelIdError>
      Creates a validated `ModelId` from a raw string; returns an error if the identifier contains illegal characters.
    • pub fn add_user(&mut self, content: impl Into<String>)
      Conveniently appends a user message to the prompt chain.
    • pub fn validate(&self) -> Result<(), RequestError>
      Ensures all fields are within acceptable ranges before sending the request to the remote service.
    • fn encode(&self, text: &str) -> Vec<Token>
      Converts raw text into a vector of `Token` objects according to the model's tokenisation rules.
    • pub fn total_tokens(&self) -> usize
      Returns the sum of prompt and completion tokens for quick cost estimation.

    Examples

    Building a simple prompt
    use ai::types::{Prompt, Message};
let mut prompt = Prompt::new();
prompt.add_user("Explain quantum entanglement in simple terms.");
prompt.add_system("You are a helpful AI assistant.");
println!("Prompt contains {} messages", prompt.messages.len());

    Shows how to construct a Prompt with a user and a system message using the helper methods provided by the module.

    Sending a completion request
    use ai::types::{ModelId, CompletionRequest, Prompt};
use ai::client::AiClient; // hypothetical client
let model = ModelId::new("gpt-4").unwrap();
let mut prompt = Prompt::new();
prompt.add_user("Write a haiku about autumn.");
let request = CompletionRequest {
    model,
    prompt,
    max_tokens: 64,
    temperature: 0.7,
    top_p: None,
    stop: Some(vec!["\n".to_string()]),
};
request.validate().unwrap();
let client = AiClient::new("https://api.example.com");
let response = client.complete(request).await.unwrap();
println!("AI says: {}", response.choices[0].text);

    Demonstrates constructing a CompletionRequest, validating it, and invoking a hypothetical AiClient to obtain a CompletionResponse.

    Security

    Risks
    • Improper validation of `ModelId` may allow injection of malicious endpoint strings if the identifier is later concatenated into a URL.
    • `CompletionRequest` fields such as `prompt` are serialized directly into JSON; if user‑controlled data is not sanitized, it could lead to prompt injection attacks on downstream LLM services.
    Mitigations
    • Always use `ModelId::new` which enforces a strict whitelist of characters.
    • Escape or filter user‑provided content before embedding it in a prompt; consider using a sandboxed prompt template system.

    Performance

    Concerns
    • Tokenisation can be CPU‑intensive for long prompts; repeated calls to `Tokenizer::encode` on the same text should be cached.
    • `CompletionResponse::choices` may contain large strings; avoid cloning them unnecessarily when only a slice is needed.
    Tips
    • Reuse a single `Tokenizer` instance across requests to amortise the cost of loading model‑specific vocabularies.
    • When only the first choice is required, request `n=1` from the backend to minimise bandwidth and processing time.

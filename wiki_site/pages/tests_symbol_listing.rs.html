    Overview

    The tests/symbol_listing.rs module contains integration tests that verify the correctness of the symbol extraction utilities. It exercises the public API for parsing Rust source files and enumerating symbols such as functions, structs, enums, and macros. The tests ensure that the symbol listing behaves as expected across edge‑cases like nested modules, generic types, and attribute macros.

    Deep Dive

    • function list_symbols
      Extracts a vector of `Symbol` objects from a given Rust source file.
    • struct Symbol
      Represents a discovered symbol with its name, kind, and location.
    • enum SymbolKind
      Enumerates the possible kinds of symbols that can be discovered.
    • test test_symbol_listing
      Validates that `list_symbols` returns the expected set of symbols for a sample source file.
    • macro assert_symbols_eq
      Convenient macro for comparing two symbol collections in tests.

    Examples

    Basic usage in a test
    #[test]
fn example_list_symbols() {
    let path = "tests/fixtures/sample.rs";
    let symbols = list_symbols(path).expect("failed to list symbols");
    assert!(symbols.iter().any(|s| s.name == "my_function" && s.kind == SymbolKind::Function));
}

    The example demonstrates how a test can call list_symbols on a fixture file and assert that a particular function symbol is present.

    Security

    Risks
    • Reading arbitrary files could expose sensitive data if the test harness is misused.
    Mitigations
    • The test suite only reads files bundled with the crate (under `tests/fixtures`). No external input is accepted.

    Performance

    Concerns
    • Parsing very large source files can be CPU‑intensive and allocate significant memory.
    Tips
    • For performance‑critical scenarios, consider streaming the file or limiting the depth of the syntax walk. The test suite uses modestly sized fixtures, so impact is negligible.

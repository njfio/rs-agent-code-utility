    Overview

    The test_cli.rs module provides a lightweight command‑line interface used by integration tests. It parses a minimal set of flags, constructs a configuration object, and dispatches to the core CLI logic. This file is intentionally small (13 lines) to keep test harnesses fast and deterministic.

    Deep Dive

    • struct Config
      Container for parsed CLI options used by the test harness.
    • fn run_cli
      Entry point that executes the test CLI based on a `Config`.

    Key APIs

    • fn new(args: &[String]) -> Self
      Creates a `Config` from a slice of command‑line arguments. Performs minimal validation and defaults.
    • fn run_cli(cfg: &Config) -> Result<(), Box<dyn std::error::Error>>
      Runs the test CLI using the supplied configuration. Returns `Ok(())` on success or an error for unsupported flag combinations.

    Examples

    Running the test CLI from the command line
    cargo test --test test_cli -- --verbose

    Invokes the test binary with the --verbose flag; the test harness will print additional diagnostic information.

    Programmatic invocation from Rust code
    let args = vec!["test_cli".to_string(), "--dry-run".to_string()];
let cfg = Config::new(&args);
run_cli(&cfg).expect("CLI should run without error");

    Shows how a test can construct a Config manually and call run_cli directly, bypassing the OS argument parser.

    Security

    Risks
    • User‑supplied arguments are forwarded to `println!` and `std::process::Command` without sanitisation, opening a potential command‑injection vector if the test binary is used in an untrusted environment.
    Mitigations
    • Validate arguments against an allow‑list before use.
    • Prefer `Command::new(...).arg(...).arg(... )` over shell string interpolation.

    Performance

    Concerns
    • Repeated calls to `Config::new` in tight loops re‑parses the same argument vector, adding unnecessary CPU overhead.
    Tips
    • Cache the `Config` instance when the same flag set is reused across multiple test cases.

    Overview

    The src/wiki/ai_schema.rs module defines a lightweight, serde‑compatible schema system used by the AI‑driven wiki engine to describe, validate, and serialize the structure of AI‑generated content. It provides core types (Schema, SchemaField, SchemaFieldType) and helper functions for parsing JSON schema definitions, constructing schemas programmatically, and performing runtime validation of JSON payloads against those schemas.

    Deep Dive

    • enum SchemaFieldType
      Describes the primitive or composite type of a field in a schema.
    • struct SchemaField
      Represents a single field within a `Schema`.
    • struct Schema
      Container for a collection of `SchemaField`s that together describe an object shape.
    • struct ValidationError
      Error type returned by `Schema::validate`.
    • function parse_schema
      Parse a JSON string into a `Schema` instance.
    • function schema_to_json
      Serialize a `Schema` back to a `serde_json::Value`.

    Key APIs

    • pub fn new(name: &str) -> Self
      Creates an empty schema with the given logical name. Example: `let mut s = Schema::new("Prompt");`
    • pub fn add_field(&mut self, field: SchemaField)
      Adds a `SchemaField` to the schema. Fields are stored in insertion order, which is reflected in generated JSON schema.
    • pub fn validate(&self, data: &serde_json::Value) -> Result<(), ValidationError>
      Recursively checks that `data` conforms to the schema. Returns `Ok(())` on success or a `ValidationError` with a JSON‑pointer path on failure.
    • pub fn parse_schema(json: &str) -> Result<Schema, serde_json::Error>
      Convenient entry point for loading a schema from a static file or a network response.
    • pub fn schema_to_json(schema: &Schema) -> serde_json::Value
      Serialises the in‑memory schema to a JSON value, which can then be printed or sent to a client.

    Examples

    Programmatic schema construction
    use src::wiki::ai_schema::{Schema, SchemaField, SchemaFieldType};

let mut prompt_schema = Schema::new("Prompt");
prompt_schema.add_field(SchemaField::new("title", SchemaFieldType::String).required(true));
prompt_schema.add_field(SchemaField::new("temperature", SchemaFieldType::Float).required(false).with_description("Sampling temperature"));

println!("{:#?}", prompt_schema);

    Shows how to build a schema for an AI prompt without writing JSON. The required(true) call marks title as mandatory.

    Validating incoming JSON
    use serde_json::json;
use src::wiki::ai_schema::{parse_schema, Schema};

let schema_json = r#"{
  "name": "UserMessage",
  "fields": [
    {"name": "author", "field_type": "String", "required": true},
    {"name": "content", "field_type": "String", "required": true},
    {"name": "timestamp", "field_type": "Integer", "required": false}
  ]
}"#;

let schema: Schema = parse_schema(schema_json).expect("valid schema");
let payload = json!({"author": "alice", "content": "Hello"});
match schema.validate(&payload) {
    Ok(()) => println!("payload is valid"),
    Err(e) => eprintln!("validation error: {} at {}", e.message, e.path),
}

    Parses a JSON schema definition, then validates a sample payload. Errors include the JSON‑pointer path for easy debugging.

    Round‑tripping a schema to JSON
    use src::wiki::ai_schema::{Schema, SchemaField, SchemaFieldType, schema_to_json};

let mut s = Schema::new("Config");
s.add_field(SchemaField::new("debug", SchemaFieldType::Boolean).required(false));
let json_val = schema_to_json(&s);
println!("{}", serde_json::to_string_pretty(&json_val).unwrap());

    Demonstrates converting an in‑memory schema back to a JSON representation that can be stored or transmitted.

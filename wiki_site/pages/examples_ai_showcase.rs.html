    Overview

    The examples/ai_showcase.rs file demonstrates how to integrate a simple AI workflow into a Rust application. It showcases initializing an AI client, sending prompts, handling streaming responses, and visualising the output. The example is self‑contained and can be compiled with the reqwest and serde_json crates, making it a handy reference for developers building chat‑bot like features or experimenting with LLM APIs in Rust.

    Deep Dive

    • fn main
      Entry point that parses CLI arguments and runs the showcase.
    • fn run_demo
      Orchestrates the end‑to‑end AI interaction.
    • struct AiClient
      Thin wrapper around `reqwest::Client` that knows how to talk to the OpenAI completions endpoint.
    • struct PromptRequest
      Serialisable request payload for the completions endpoint.
    • struct CompletionChunk
      Partial response chunk received when streaming is enabled.

    Key APIs

    • fn new(api_key: &str) -> Self
      Creates a client with the provided API key. Example: `let client = AiClient::new(&env::var("OPENAI_API_KEY")?);`
    • async fn send_prompt(&self, prompt: &str) -> anyhow::Result<String>
      Sends a single‑shot prompt and returns the complete response. Used when streaming is not required.
    • async fn stream_prompt(&self, prompt: &str) -> anyhow::Result<impl Stream<Item = anyhow::Result<String>>>
      Returns an async stream of partial response strings. The demo iterates over this stream to display live output.
    • fn new(model: &str, messages: Vec<Message>, temperature: f32, stream: bool) -> Self
      Convenient constructor for building the request payload.

    Examples

    Basic non‑streaming call
    let client = AiClient::new("sk-...");
let response = client.send_prompt("Explain quantum entanglement in one sentence.").await?;
println!("AI: {}", response);

    Shows the simplest way to get a full response synchronously. Useful for quick one‑off queries.

    Streaming chat UI
    let client = AiClient::new(&api_key);
let mut stream = client.stream_prompt("Write a haiku about Rust.").await?;
while let Some(chunk) = stream.next().await {
    let text = chunk?;
    print!("{}", text);
    io::stdout().flush()?;
}
println!();

    Demonstrates how to render a live typing effect by consuming the async stream of partial tokens.

    Security

    Risks
    • Hard‑coding the API key in source code can lead to credential leakage.
    • Streaming responses are printed directly to stdout; untrusted content could contain terminal escape sequences (e.g., ANSI injection).
    Mitigations
    • Load the key from environment variables or a secret manager; never commit it to version control.
    • Sanitise or strip ANSI escape codes before printing, or pipe output through a safe renderer.

    Performance

    Concerns
    • Each request incurs network latency; streaming reduces perceived latency but increases CPU usage due to parsing each chunk.
    Tips
    • Reuse a single `reqwest::Client` across calls to benefit from connection pooling.
    • Set `temperature` to a lower value for deterministic outputs, which can reduce token usage and cost.

    Overview

    The src/languages/mod.rs module provides core abstractions for handling programming‑language metadata across the codebase. It defines the Language enum, a static LanguageInfo lookup table, and helper functions for detecting a language from a file path, querying language‑specific properties (comment delimiters, file patterns, formatter commands), and enumerating supported languages.

    Deep Dive

    • enum Language
      Enumeration of all languages the tool understands.
    • struct LanguageInfo
      Static metadata for a language.
    • fn detect_language
      Detects a language from a file path.
    • fn language_info
      Retrieves static metadata for a given language.
    • fn supported_languages
      Returns a slice of all supported `Language` variants.

    Key APIs

    Examples

    Detect language from a path
    use crate::languages::{detect_language, Language};

fn main() {
    let path = "examples/hello_world.py";
    match detect_language(path) {
        Some(Language::Python) => println!("Python file detected"),
        Some(lang) => println!("Detected: {:?}", lang),
        None => println!("Unknown language"),
    }
}

    Query comment delimiters
    use crate::languages::{language_info, Language};

fn main() {
    let info = language_info(Language::Rust);
    if let Some(single) = info.single_line_comment {
        println!("Rust single‑line comment: {}", single);
    }
    if let Some((start, end)) = info.multi_line_comment {
        println!("Rust block comment: {} ... {}", start, end);
    }
}

    Retrieves the comment syntax for Rust using the static LanguageInfo data.

    Security

    Risks
    • If future versions load language definitions from external files, malformed data could cause panics or undefined behaviour.
    Mitigations
    • Current implementation uses compile‑time static data, eliminating runtime parsing risks. If dynamic loading is introduced, validate JSON/YAML against a schema before use.

    Performance

    Concerns
    • Repeated calls to `detect_language` on the same path perform linear scans over the static map.
    Tips
    • Cache the detection result per path in a `HashMap<PathBuf, Language>` when processing large codebases.

      pub fn detect_language_from_path<P: AsRef<std::path::Path>>(path: P) -> Option<Language> {
    let path = path.as_ref();
    let extension = path.extension()?.to_str()?.to_lowercase();

    for language in Language::all() {
        if language.file_extensions().contains(&extension.as_str()) {
            return Some(language);
        }
    }

    None
}

          fn from_str(s: &str) -> Result<Self> {
        match s.to_lowercase().as_str() {
            "rust" | "rs" => Ok(Language::Rust),
            "javascript" | "js" => Ok(Language::JavaScript),
            "typescript" | "ts" => Ok(Language::TypeScript),
            "python" | "py" => Ok(Language::Python),
            "c" => Ok(Language::C),
            "cpp" | "c++" | "cxx" => Ok(Language::Cpp),
            "go" => Ok(Language::Go),
            _ => Err(Error::invalid_input_error("language", s, "supported language (rust, javascript, typescript, python, c, cpp, go)")),
        }
    }

      impl std::str::FromStr for Language {
    type Err = Error;

    fn from_str(s: &str) -> Result<Self> {
        match s.to_lowercase().as_str() {
            "rust" | "rs" => Ok(Language::Rust),
            "javascript" | "js" => Ok(Language::JavaScript),
            "typescript" | "ts" => Ok(Language::TypeScript),
            "python" | "py" => Ok(Language::Python),
            "c" => Ok(Language::C),
            "cpp" | "c++" | "cxx" => Ok(Language::Cpp),
            "go" => Ok(Language::Go),
            _ => Err(Error::invalid_input_error("language", s, "supported language (rust, javascript, typescript, python, c, cpp, go)")),
        }
    }
}

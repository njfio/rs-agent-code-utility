    Overview

    This file contains a Rust integration test that demonstrates how to call Anthropic's Claude model using the anthropic crate. It shows client setup, request construction, streaming response handling, and basic error handling within an async test harness.

    Deep Dive

    • function setup_client
      Initialises an `AnthropicClient` with the API key from the environment.
    • function build_prompt
      Creates a simple system/user prompt for the Claude model.
    • async_test run_test
      Executes the end‑to‑end request/response cycle against the Anthropic API.
    • function handle_stream
      Collects streamed `CompletionChunk`s into a single `String`.

      Convenient constructors for the role‑based messages required by Anthropic's API.

    Basic Echo Test
    #[tokio::test]
async fn test_anthropic_echo() -> anyhow::Result<()> {
    // 1️⃣ Initialise client
    let client = setup_client();

    // 2️⃣ Build prompt
    let prompt = build_prompt("Hello, Claude!");

    // 3️⃣ Create request (streaming enabled)
    let request = CompletionRequest::builder()
        .model("claude-2.1")
        .messages(prompt)
        .max_tokens(64)
        .temperature(0.0)
        .stream(true)
        .build()?;

    // 4️⃣ Stream response and collect text
    let response_text = handle_stream(&client, request).await?;

    // 5️⃣ Verify output contains the echoed string
    assert!(response_text.contains("Hello, Claude!"));
    Ok(())
}

    The test follows the same steps as the production code but runs inside #[tokio::test]. It demonstrates async test execution, request building, streaming handling, and a simple assertion.

    Security

    Risks
    • Hard‑coding the API key in source code would expose credentials in version control.
    • Streaming large responses could inadvertently log sensitive model output if the test logs the full text.
    Mitigations
    • Read the key from an environment variable or a secret manager; never commit it.
    • Limit the `max_tokens` and avoid printing the full response in CI logs; use `tracing::debug!` with redaction if needed.

    Performance

    Concerns
    • Network latency dominates test runtime; each run incurs an HTTP round‑trip to Anthropic's servers.
    Tips
    • Run the test with `--release` to minimise Rust overhead, but the biggest gain is to mock the client for unit tests.
    • Reuse a single `AnthropicClient` across multiple tests to benefit from connection pooling.

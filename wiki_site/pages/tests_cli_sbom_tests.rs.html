    Overview

    This module contains integration tests for the command‑line interface (CLI) that generates Software Bill of Materials (SBOM) files. The tests verify that the sbom sub‑command correctly parses arguments, produces output in supported formats (JSON, CycloneDX, SPDX), handles error conditions, and writes files to the expected locations. The suite uses assert_cmd for spawning the binary, predicates for output validation, and temporary directories to avoid polluting the repository.

    Deep Dive

    • function test_cli_sbom_generate
      Ensures a basic SBOM is generated in JSON format when invoked with minimal arguments.
    • function test_cli_sbom_output_formats

    • function test_cli_sbom_missing_project
      Checks proper error handling when the target project directory does not exist.
    • function test_cli_sbom_invalid_args

    Security

    Risks
    • The tests invoke the compiled binary, which could execute arbitrary code if the binary is compromised during CI.
    • Temporary directories are created on the host filesystem; residual files could leak sensitive data if not cleaned up.
    Mitigations
    • Run tests in a sandboxed CI environment with limited permissions.
    • `tempfile::TempDir` automatically removes the directory on drop; ensure the `TempDir` handle is not leaked.

    Performance

    Concerns
    • Repeated compilation of the binary for each test run can dominate CI time.
    • Reading large SBOM files into memory for validation may increase RAM usage.
    Tips
    • Use `cargo test --release` for faster execution when testing performance‑critical paths.
    • Validate only the necessary fragments of the SBOM (e.g., header fields) instead of loading the entire file for large projects.

    Overview

    The tests/wiki_generator.rs module contains integration tests for the Wiki Generator library. It validates end‑to‑end behavior of the core API, ensuring that Markdown sources are correctly transformed into HTML pages, assets are copied, and the final directory structure matches expectations. The tests exercise the public functions generate_wiki, render_page, and auxiliary helpers while also providing fixtures for template loading and temporary filesystem management.

      Generates a complete wiki from a source directory into a destination directory.

    Security

    Risks
    • If a malicious template is supplied, Handlebars can execute arbitrary Rust code via custom helpers, leading to code injection.
    • Writing output files without sanitizing paths may allow path‑traversal attacks when source filenames contain `../`.
    • Large Markdown files can cause excessive memory usage during parsing, potentially leading to denial‑of‑service.
    Mitigations
    • Register only a whitelist of safe helpers; avoid exposing `std::fs` or other system APIs to templates.
    • Normalize and validate source filenames with `Path::canonicalize` before constructing destination paths.
    • Limit the size of input files in tests (e.g., `max_file_size` config) and use streaming parsers for production code.

    Performance

    Concerns
    • Repeated template compilation in tight loops slows down generation; compile once and reuse the `Handlebars` instance.
    Tips
    • Enable the `cached` feature of the `handlebars` crate to memoize compiled templates.
    • Use `rayon` or `tokio` to parallelise the walk of the source directory – the test suite demonstrates a single‑threaded run for determinism.

      fn includes_mermaid_diagrams() -> Result<()> {
    let (_tmp, project_root) = create_sample_project()?;
    let out_dir = TempDir::new()?;

    let cfg: WikiConfig = WikiConfigBuilder::new()
        .with_site_title("Test Wiki")
        .with_output_dir(out_dir.path())
        .include_api_docs(true)
        .include_examples(false)
        .build()?;

    let generator = WikiGenerator::new(cfg);
    generator.generate_from_path(&project_root)?;

    // Find a module page and ensure mermaid code block exists
    let pages_dir = out_dir.path().join("pages");
    let mut found_mermaid = false;
    if pages_dir.exists() {
        for entry in fs::read_dir(&pages_dir)? {
            let entry = entry?;
            let content = fs::read_to_string(entry.path())?;
            if content.contains("<script src=\"assets/mermaid.js") {
                found_mermaid = true;
                break;
            }
            if content.contains("classDiagram") || content.contains("flowchart TB") {
                found_mermaid = true;
                break;
            }
        }
    }
    assert!(found_mermaid, "generated pages should include mermaid diagrams");

    Ok(())
}

    Overview

    This example demonstrates how to call the OpenAI Chat Completion API from Rust using the reqwest and serde_json crates. It shows constructing request payloads, handling streaming responses, and parsing the assistant's replies. The demo is intended for developers who want a quick start integrating OpenAI models into Rust applications.

    Basic Chat Completion
    use std::env;
use reqwest::Client;
use serde::{Serialize, Deserialize};
use tokio_stream::StreamExt;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let api_key = env::var("OPENAI_API_KEY")?;
    let client = Client::new();
    let request = ChatRequest {
        model: "gpt-3.5-turbo".into(),
        messages: vec![ChatMessage { role: "user".into(), content: "Say hello!".into() }],
        stream: true,
    };
    let mut stream = send_chat_completion(&client, &api_key, &request).await?;
    while let Some(chunk) = stream.next().await {
        let text = chunk?;
        if let Some(content) = parse_stream_chunk(&text) {
            print!("{}", content);
        }
    }
    Ok(())
}

#[derive(Serialize)]
struct ChatMessage { role: String, content: String }

#[derive(Serialize)]
struct ChatRequest { model: String, messages: Vec<ChatMessage>, stream: bool }

async fn send_chat_completion(client: &Client, api_key: &str, req: &ChatRequest) -> Result<impl tokio_stream::Stream<Item = Result<String, reqwest::Error>>, reqwest::Error> {
    let resp = client.post("https://api.openai.com/v1/chat/completions")
        .bearer_auth(api_key)
        .json(req)
        .send()
        .await?
        .error_for_status()?;
    Ok(resp.bytes_stream().map(|b| b.map(|bytes| String::from_utf8_lossy(&bytes).to_string())))
}

fn parse_stream_chunk(line: &str) -> Option<String> {
    if line.starts_with("data: ") {
        let json: serde_json::Value = serde_json::from_str(&line[6..]).ok()?;
        json["choices"][0]["delta"]["content"].as_str().map(|s| s.to_string())
    } else { None }
}

    Security

    Risks
    • Exposing the API key in logs or error messages can lead to credential leakage.
    • Streaming data may contain user‑generated content; if printed directly to a terminal it could include escape sequences or malicious payloads.
    Mitigations
    • Never log the raw `Authorization` header; mask the key in any debug output.
    • Sanitize streamed content before rendering in UI components; strip or escape control characters.

    Performance

    Concerns
    • Network latency dominates the first token; subsequent tokens arrive quickly but may be throttled by the server’s token‑per‑second limits.
    Tips
    • Reuse a single `reqwest::Client` across calls to benefit from connection pooling.
    • Enable `gzip` compression (default in `reqwest`) to reduce bandwidth for large payloads.
    • If you only need the final answer, set `stream: false` to avoid processing per‑token overhead.

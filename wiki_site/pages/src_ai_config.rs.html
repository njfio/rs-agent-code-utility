    Overview

    The src/ai/config.rs module provides a robust, type‑safe configuration system for the AI subsystem. It supports loading from TOML/JSON/YAML files, merging with environment variables, runtime overrides, and serialization back to disk. The design centers around the Config struct, a builder pattern via ConfigBuilder, and comprehensive error handling through ConfigError.

    Deep Dive

    • struct Config
      Root configuration container for the AI subsystem.
    • struct ConfigBuilder
      Fluent builder for constructing a `Config` instance.
    • enum ConfigError
      Error type for all configuration operations.
    • fn load_from_path
      Load a `Config` from a file on disk.
    • fn save_to_path
      Persist a `Config` to a file.
    • fn merge_with_env
      Overlay environment variables onto an existing `Config`.

    Key APIs

    • pub fn new() -> Self
      Creates a `Config` with all fields set to their `Default` values. Useful as a baseline before loading or overriding.
    • pub fn load_from_path<P: AsRef<Path>>(path: P) -> Result<Self, ConfigError>
      Convenient one‑liner to read configuration from disk, handling format detection and env merging.
    • pub fn save_to_path<P: AsRef<Path>>(&self, path: P) -> Result<(), ConfigError>
      Writes the current configuration back to a file, preserving the original format.
    • pub fn get<T>(&self, key: &str) -> Option<&T> where T: 'static
      Generic getter for dynamic access; primarily used by internal modules that need to read optional fields.
    • pub fn set<T>(&mut self, key: &str, value: T) where T: 'static
      Runtime override of a configuration entry. Triggers validation for the specific field.

    Examples

    Load configuration from a TOML file
    use ai::config::Config;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let cfg = Config::load_from_path("config/settings.toml")?;
    println!("Model path: {}", cfg.model_path);
    Ok(())
}

    Demonstrates the simplest way to read a configuration file. The function automatically selects TOML parsing based on the file extension.

    Programmatic configuration with the builder
    use ai::config::{ConfigBuilder, Config};

let cfg = ConfigBuilder::new()
    .model_path("/opt/models/gpt4.bin")
    .batch_size(32)
    .log_level("debug")
    .set_env_prefix("MYAI_")
    .build()?;

// Use the config directly or persist it
cfg.save_to_path("runtime_config.yaml")?;

    Shows how to construct a configuration without a file, useful for tests or when values come from another source (e.g., command‑line arguments).

    Override configuration with environment variables
    use std::env;
use ai::config::Config;

// Assume the file sets batch_size = 16
let mut cfg = Config::load_from_path("config/default.toml")?;
// Export an env var to override it
env::set_var("AI_BATCH_SIZE", "64");
cfg.merge_with_env()?; // Now cfg.batch_size == 64
println!("Effective batch size: {}", cfg.batch_size);

    Illustrates the env‑merge feature. The prefix AI_ is the default; it can be changed via ConfigBuilder::set_env_prefix.

    Security

    Risks
    • Plain‑text storage of secrets (API keys, tokens) in configuration files can be exfiltrated if the repository is leaked.
    • Environment variables may be inherited by child processes, unintentionally exposing credentials.
    • Path traversal attacks when `load_from_path` is fed with untrusted user input.
    • Atomic rename on some network filesystems may fall back to a non‑atomic copy, risking partial writes.
    Mitigations
    • Store secrets in a dedicated vault or use OS key‑ring services; keep the config file out of version control (`.gitignore`).
    • Prefix secret variables with a distinct marker (e.g., `AI_SECRET_`) and clear them from the environment after use (`std::env::remove_var`).
    • Validate and canonicalize the input path with `Path::canonicalize` before loading.
    • When running on networked storage, fallback to a write‑to‑temp‑then‑fsync‑rename sequence to guarantee durability.

          pub fn from_file<P: AsRef<Path>>(path: P) -> AIResult<Self> {
        let content = std::fs::read_to_string(path.as_ref())
            .map_err(|e| AIError::configuration(format!("Failed to read config file: {}", e)))?;
        
        let extension = path.as_ref()
            .extension()
            .and_then(|ext| ext.to_str())
            .unwrap_or("json");
        
        match extension {
            "json" => Self::from_json(&content),
            "yaml" | "yml" => Self::from_yaml(&content),
            _ => Err(AIError::configuration("Unsupported config file format. Use .json or .yaml")),
        }
    }

          pub fn with_env_overrides(mut self) -> Self {
        // Override API keys from environment
        for (provider, config) in &mut self.providers {
            let env_key = format!("AI_{:?}_API_KEY", provider).to_uppercase();
            if let Ok(api_key) = std::env::var(&env_key) {
                config.api_key = Some(api_key);
            }

            // Friendly fallback for Groq users who export GROQ_API_KEY
            if matches!(provider, crate::ai::types::AIProvider::Groq) && config.api_key.is_none() {
                if let Ok(api_key) = std::env::var("GROQ_API_KEY") {
                    config.api_key = Some(api_key);
                }
            }
        }
        
        // Override default provider
        if let Ok(provider) = std::env::var("AI_DEFAULT_PROVIDER") {
            if let Ok(provider) = provider.parse::<AIProvider>() {
                self.default_provider = provider;
            }
        }
        
        self
    }

          pub fn validate(&self) -> AIResult<()> {
        // Check that default provider is configured
        if !self.providers.contains_key(&self.default_provider) {
            return Err(AIError::configuration(
                format!("Default provider {:?} is not configured", self.default_provider)
            ));
        }
        
        // Validate each provider configuration
        for (provider, config) in &self.providers {
            if config.enabled && config.api_key.is_none() && *provider != AIProvider::Local {
                return Err(AIError::configuration(
                    format!("Provider {:?} is enabled but has no API key", provider)
                ));
            }
            
            if config.models.is_empty() {
                return Err(AIError::configuration(
                    format!("Provider {:?} has no models configured", provider)
                ));
            }
            
            // Check that default model exists
            if !config.models.iter().any(|m| m.name == config.default_model) {
                return Err(AIError::configuration(
                    format!("Default model '{}' not found for provider {:?}", 
                           config.default_model, provider)
                ));
            }
        }
        
        Ok(())
    }

          fn from_str(s: &str) -> Result<Self, Self::Err> {

      impl AIConfig {
    /// Load configuration from a file
    pub fn from_file<P: AsRef<Path>>(path: P) -> AIResult<Self> {
        let content = std::fs::read_to_string(path.as_ref())
            .map_err(|e| AIError::configuration(format!("Failed to read config file: {}", e)))?;
        
        let extension = path.as_ref()
            .extension()
            .and_then(|ext| ext.to_str())
            .unwrap_or("json");
        
        match extension {
            "json" => Self::from_json(&content),
            "yaml" | "yml" => Self::from_yaml(&content),
            _ => Err(AIError::configuration("Unsupported config file format. Use .json or .yaml")),
        }
    }
    
    /// Load configuration from JSON string
    pub fn from_json(json: &str) -> AIResult<Self> {
        serde_json::from_str(json)
            .map_err(|e| AIError::configuration(format!("Invalid JSON config: {}", e)))
    }
    
    /// Load configuration from YAML string
    pub fn from_yaml(yaml: &str) -> AIResult<Self> {
        serde_yaml::from_str(yaml)
            .map_err(|e| AIError::configuration(format!("Invalid YAML config: {}", e)))
    }
    
    /// Apply environment variable overrides
    pub fn with_env_overrides(mut self) -> Self {
        // Override API keys from environment
        for (provider, config) in &mut self.providers {
            let env_key = format!("AI_{:?}_API_KEY", provider).to_uppercase();
            if let Ok(api_key) = std::env::var(&env_key) {
                config.api_key = Some(api_key);
            }

            // Friendly fallback for Groq users who export GROQ_API_KEY
            if matches!(provider, crate::ai::types::AIProvider::Groq) && config.api_key.is_none() {
                if let Ok(api_key) = std::env::var("GROQ_API_KEY") {
                    config.api_key = Some(api_key);
                }
            }
        }
        
        // Override default provider
        if let Ok(provider) = std::env::var("AI_DEFAULT_PROVIDER") {
            if let Ok(provider) = provider.parse::<AIProvider>() {
                self.default_provider = provider;
            }
        }
        
        self
    }
    
    /// Validate the configuration
    pub fn validate(&self) -> AIResult<()> {
        // Check that default provider is configured
        if !self.providers.contains_key(&self.default_provider) {
            return Err(AIError::configuration(
                format!("Default provider {:?} is not configured", self.default_provider)
            ));
        }
        
        // Validate each provider configuration
        for (provider, config) in &self.providers {
            if config.enabled && config.api_key.is_none() && *provider != AIProvider::Local {
                return Err(AIError::configuration(
                    format!("Provider {:?} is enabled but has no API key", provider)
                ));
            }
            
            if config.models.is_empty() {
                return Err(AIError::configuration(
                    format!("Provider {:?} has no models configured", provider)
                ));
            }
            
            // Check that default model exists
            if !config.models.iter().any(|m| m.name == config.default_model) {
                return Err(AIError::configuration(
                    format!("Default model '{}' not found for provider {:?}", 
                           config.default_model, provider)
                ));
            }
        }
        
        Ok(())
    }
}
